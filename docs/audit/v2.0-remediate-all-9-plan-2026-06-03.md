## Tier 1 — additive lockstep fixes (target 93.53% → ~95–96%, DMA stays green)

Each is a localized, feature-gated change validated like the prior PPU ports: default byte-identical,
60-ROM oracle 60/60, DMA cluster green, all canaries green, then promote to default.

### T1.1 — `$2002 flag timing` [error 1]  (PPU-only)
Four `$2002` reads walk the pre-render dot-1 flag-clear; HW latches **VBL (bit7) at read-start
(M2 high)** but samples **sprite-0 (bit6)/overflow (bit5) ~1.875 dots later at read-end (M2 low)**.
main collapses all three to one cycle-start sample (`crates/nes-ppu/src/ppu.rs` `cpu_read_register`
$2002 arm ~`:750`). Fix (TriCNES `Emulator.cs:939-947` `EmulateUntilEndOfRead`): the read owns a
two-point sample. **See the execution log below for the refined, lockstep-safe COMPUTED design.**

### T1.2 — `Delta Modulation Channel` [error 21, fails Test L]  (APU)
Test L places a `$4015` DMC-enable write 0/1/2 cycles before the DMC timer hits 0 and checks the
3-vs-4-cycle DMA span. main uses a fixed `apu_phase`-based `dmc_dma_delay` (`crates/nes-apu/src/apu.rs:667`).
Reference: ares `fc/apu/dmc.cpp:7`. **See the execution log — RE-CLASSIFIED (DMA-coupled).**

### T1.3 — `NMI Overlap BRK` [error 2]  (CPU-only)
BRK needs no DMC arming, so it's the one C1-trio test closable on the lockstep. Port the canonical
recognition state-machine (Mesen2 `NesCpu.cpp:445-464`/`:531-551`, TriCNES `Emulator.cs:4392-4488`):
delayed `prev_need_nmi` hijack, I-mask before the 7-cycle service, cycle-5 live-`need_nmi` vector,
BRK `prev_need_nmi` clear — replacing main's `nmi_first_tick` approximation (`crates/nes-cpu/src/cpu.rs`
`service_interrupt` ~`:699-714`). **Risk:** M2-high vs M2-low sample position is coupled to blargg
#4/#5 (memory `project_c1_trace_loop_ceiling.md` §29) — keep M2-high, change only dispatch/hijack/
deferral, re-run the full `cpu_interrupts_v2` 1–5 gauntlet.

## Tier-1 execution log — iteration 1 (2026-06-03)

First pass of the Tier-1 loop (analysis + close implementation reads; **no code changes committed**
— the subtle timing edits were deferred to a focused empirical session to protect the 93.53% /
DMA-green / 60-ROM-oracle baseline). Two plan-refining findings:

### Finding A — T1.2 is DMA-COUPLED, not cleanly additive (RE-CLASSIFIED, deprioritized)
The Test L/M/N pass condition is the **3-vs-4-cycle span**, which is set by `service_dmc_dma`'s
**get/put-phased alignment** in `bus.rs` (noop_cycles + alignment) — **not** by `dmc_dma_delay`.
The arm-delay (`apu.rs:667` `apu_phase{4}else{3}`) only positions *which* get/put phase the DMA
lands on, which then determines the span. ⇒ T1.2 couples to the same get/put axis the **DMA cluster
depends on** (real DMA risk; closer to Tier-2 DMC-cycle-exactness). Also confirmed: the abandoned
branch uses the **identical** `apu_phase{4}else{3}` model and **also fails Test L** — no working
port exists; this is novel logic. asm hint (`AccuracyCoin.asm:11201`): the enable delay is "2 or 3
cycles" vs the code's 3/4, but the span lives in the bus service, not the delay. **Next (fresh
empirical loop):** instrument the current Test-L arm cycle + get/put phase + resulting span, then
craft a get/put-coherent enable-retry in `service_dmc_dma`; validate the DMA cluster + 60-ROM oracle
at EVERY iteration (ASK before the service change). **Deprioritized below T1.1/T1.3.**

### Finding B — T1.1 design refined to a lockstep-safe COMPUTED two-point sample (READY)
A physical "advance the PPU 2 dots mid-read" double-advances (the lockstep read returns BEFORE the
cycle's 3 dots tick). The correct lockstep-safe model is a **computed** read-end sample: in
`ppu.rs cpu_read_register` $2002 arm, behind feature `ppu-2002-read-end-flags` — bit7 (VBL) =
current value (read-start); bits 6/5 (sprite-0/overflow) = value as-of ~2 dots later (read-end) via
a helper `sprite_flags_clear_imminent(READ_END_DOTS)` that returns true when advancing N dots from
`(scanline, dot)` reaches/passes pre-render(261) dot 1 (the flag-clear), in which case mask bits 6/5
to 0. Byte-identical except within ~2 dots before the clear (real games ~never read $2002 there →
oracle stays 60/60). `READ_END_DOTS ≈ 2` (TriCNES `EmulateUntilEndOfRead` = 7 mc ≈ 1.75 dot).
**Empirical tuning needed:** the exact boundary positions (candidates `(261,0)` & `(260,340)`) +
`READ_END_DOTS` — run the battery, read `$048D`, match the 4-read walk `$E0,$E0,$80,$00`; then handle
Test 2 (the SET edge) if the error advances past 1. DMA-free (PPU-only) ⇒ **the safest Tier-1 fix to
land first.** Guards each iteration: $2002 flag timing flips, 60-ROM oracle 60/60, ppu_vbl_nmi 10/10,
visual_regression 7/7, nes-ppu 41/41.
