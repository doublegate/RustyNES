**Step 3 — `$2002 flag timing` (Tier 1.1, 2026-06-03):** novel logic (no clean
port exists — the branch couples this to the OAM-span/master-clock axis), built
from the TriCNES `EmulateUntilEndOfRead` two-point model. `$2002` latches VBL (bit 7)
at read-start (M2 high) but samples the unlatched sprite-0/overflow (bits 6/5) at
read-end (~1.875 dots later, M2 low). Implemented as a **computed** read-end sample
in `ppu.rs cpu_read_register` (lockstep-safe — a physical mid-read PPU advance would
double-advance): a helper `sprite_flags_clear_imminent(dots)` masks bits 6/5 on the
**returned byte only** when advancing `STATUS_READ_END_DOTS` dots reaches/passes the
pre-render dot-1 flag-clear. Tuning: **`STATUS_READ_END_DOTS = 1`** — the test's reads
are 1 PPU dot apart, so a 1-dot window yields the primary key `$E0,$E0,$80,$00`; a
2-dot window also passes Test 1 (via the alt key `$E0,$80,$80,$00`) but masks one
extra read position, which regressed `$2004 Stress` (caught immediately). Open-bus
I/O-latch refresh deliberately left on the read-start value so the decay model is
byte-identical for real games. Both `$2002` sub-tests (flag-clear + flag-set) pass.
Verified: 94.24%, DMA cluster green, default byte-identical pre-promotion,
`cpu_interrupts_v2` 5/5, `ppu_vbl_nmi` 10/10, `nes-ppu` 41/41, `visual_regression`
7/7, fmt/clippy/no_std clean. **60-ROM oracle 60/60 with ONE deliberate re-baseline:**
Ganbare Goemon (VRC1) reads `$2002` within 1 dot of the pre-render clear and consumes
the returned sprite bits in its audio/RNG path → its `audio_fnv1a64` shifted
(framebuffer hash + cycle count byte-identical). The new value is hardware-faithful
(the AccuracyCoin `$2002 flag timing` test, by the TriCNES answer-key author, proves
real HW returns this masked byte); the prior golden captured the less-accurate
single-point read. User-approved re-baseline (framebuffer unchanged ⇒ no visual
regression, per `feedback_paired_pipeline_oracle_masking`). **Promoted to default.**

## T1.3 NMI Overlap BRK — investigated + REFUTED on the lockstep (2026-06-03, step 4)

**T1.3 is NOT closable on the lockstep pivot.** Two empirical results, no code committed:

1. **CPU-only attempt is insufficient.** `NMI Overlap BRK` is the canonical interrupt-hijacking
   sweep: NMI fired across 32 one-cycle alignments vs a `BRK`, checked against a 64-byte answer
   key (2 keys for the 2 power-on phases). Currently error 2 = Test 1 (BRK skips operand) passes,
   Test 2 (per-alignment pushed-P/vector) fails. The Mesen2 model (`NesCpu.cpp` edge-latched
   `_needNmi`/`_prevNeedNmi`, BRK live-`_needNmi` vector + `_prevNeedNmi=false`) needs the
   NMI-set dot to align with the CPU's per-cycle sample phase — the integer-dot ceiling.

2. **The R2 "DMA-safe shortcut" is refuted.** Hypothesis: `cpu-c1-attempt-17-access-reorder`
   (φ1/φ2 split = "R1-lite") + the R2 on-time PPU predicates (`acddd22`: NMI raise dot 3→1,
   `$2002` drops /NMI unconditionally, race window dot==0) would close the C1 trio DMA-safely.
   **Built it behind `mc-r2-ontime-ppu-events` and probed — FAILED on both counts:**
   - C1 trio still fails (nmi_and_brk, nmi_and_irq), AccuracyCoin **dropped to 92.81%** (worse).
   - **attempt-17 is NOT DMA-safe at the interaction level**: blargg `cpu_interrupts_v2/4-irq_and_dma`
     REGRESSES (passes on default), even though the AccuracyCoin "APU Registers and DMA" suite
     stays 10/0. So attempt-17 ≠ a clean R1 stand-in — R1's "double catch-up" runs the PPU to the
     access's **exact mc**, more than attempt-17's fixed 1-dot φ1, and the closure also needs the
     CPU-side A2 recognition (`24b7ca0`/`a69f1e2`) + R3 cold-boot `mc+=12` (`project_c1_trace_loop_ceiling`).
   - **Reverted** (default-off probe, no benefit — same rationale as the `ppu-2007-read-buffer` revert).

**Conclusion (confirms the original BLOCKED call):** the C1 trio (BRK + IRQ + flag latency) needs
the **full master-clock R1 substrate ported DMA-safely** — the deferred multi-week core
(`project_r1_dma_regression` / `project_t_track_dmc_integration`), the exact axis the pivot exists
to avoid. There is no lockstep shortcut. R2's PPU predicates are correct ONLY on the real R1
substrate; bolted onto attempt-17 they regress DMA + don't close the trio.
