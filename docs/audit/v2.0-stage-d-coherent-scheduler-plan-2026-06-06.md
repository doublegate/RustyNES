## Why this is the only remaining path (proven this session)

`docs/audit/v2.0-phase1a-impl-abort-arm-2026-06-06.md` proved: the abort ARM is
bounded+solved (`timer ∈ {10,8}`), but the upstream gate `CheckDMATiming Y==4`
(reload-DMA span 3→4) is **coupled to the C1 IRQ trio** — `CheckDMATiming` /
`DMASync` / the abort sweep all anchor on the same looping reload DMA, so a
reload-only span lever is inert or starves the GET; making the reload genuinely 4
cycles needs OAM on the unified counter (`RW-2`), which regresses C1 unless IRQ φ2
sampling is also coherent on that counter. That simultaneous requirement = the
named **C1-vs-DMA conflict**, closeable only by the coherent scheduler below.

## Target architecture (DESIGN-agent scoped)

`master_clock` sole authority; `idle_tick` → `internal_cycle` (a dummy read of the
held bus address = Mesen `DummyRead`); `read1`/`write1`/`internal_cycle` the ONLY
cycle advancers (DMA-halt then automatic every cycle → the `mc-r1-dma-cycle-budget`
+ `mc-r1-dmc-idle-halt` hacks DELETED); OAM DMA promoted to the same CPU-driven
per-cycle interleave the DMC already uses under R1 (retire `drain_dma` /
`clock_oam_dma_cycle` burst).

- **Stage A** — promote `cpu-implied-dummy-reads` (pre-written; cycle-2 PC dummy
  reads for implied ops). Gated under the umbrella so it is abort-safe only once
  Stage D lands; verify no C1/DMA regression.
- **Stage B** — stack-op per-cycle dummy reads (`cpu-stack-dummy-reads`, already in
  the umbrella base; confirm coherent under the unified counter).
- **Stage C** — indexed-mode index/pointer dummy reads (NEW): `addr_zp_x/zp_y/ind_x`
  pre-add dummy + `abs_x/abs_y/zp` store-mode unconditional dummy (Mesen
  `GetZeroXAddr`/`GetIndXAddr`/`GetAbsXAddr`); per-cycle cell-trace gate vs Mesen.
- **Stage D (MAKE-OR-BREAK)** — OAM unify, LAST of the dummy work: each OAM cycle
  wrapped `start_cycle → unified_dma_step → end_cycle` so it samples IRQ/NMI via the
  SAME φ2 `_prev*` pipeline `RW-2` bypassed; DMC-get-before-OAM-get priority; parity
  from the one counter; `put_cycle` toggles last (TriCNES CPUClock 12→0 order). This
  is where reload-span 3→4 (`CheckDMATiming Y=4`) is won AND C1 is preserved — the
  two together. Then the SOLVED Phase-1A arm (`mc-r1-impl-abort-at-enable`,
  `timer ∈ {10,8}`) + the end-of-sample reload-fetch emit register on the Y==4 base
  → Implicit/Explicit Abort + DMC+OAM close.
- **Stage E** — delete `idle_tick`/burn-loop/`cycles_emitted`/`dma-cycle-budget`/
  OAM-burst; introduce `internal_cycle`. Tripwire `debug_assert_eq!(cycles_emitted,
  cycles)` at the old burn-loop site during migration.
- **Stage F** — promote `mc-r1-full-cpu` to default; serialize `master_clock` as the
  sole save-state authority (RW-3 blast radius); retire the `#[ignore]`'d DMA-tail
  tests; final 139/139 gauntlet.
