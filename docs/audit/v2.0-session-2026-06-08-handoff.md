## Program M — what landed, and what was REFUTED by measurement

Five gated agents drove Program M to a precise dead-reckoning. All gated default-off, NOT in the umbrella:

- `mc-r1-dmc-reload-visibility-delay` — **M-1**: reload arms into `_next` at cycle-START, promoted at
  end-of-cycle → first-service on the PUT cycle (span-4), matching Mesen `StartCpuCycle→ProcessCpuClock`
  / TriCNES `_6502→_EmulateAPU`. Flipped the Explicit `idx[7]` entry a prior audit called "IMMOVABLE".
- `mc-r1-dmc-oam-overlap` — **M-2 overlap**: ports the lockstep DMC-during-OAM exact per-event structure
  (unconditional `noop_cycles` → OAM-stalled GET → post-GET realign stall, one DMC/entry). Eliminated the
  `2C`/44 runaway; DMC+OAM values now in-range.
- `mc-r1-dmc-bytetimer-end` — byte-timer clock START→END. Refuted as a standalone closure.
- `mc-r1-counter-collapse` — **put_cycle end-flip + byte-timer on `timer_phase=!put_cycle`**. **FIRST
  CheckDMATiming Y 3→4 via put_cycle parity in 18 months**; DMC+OAM front-half == KEY (only `idx[7]` off).
  But a GLOBAL parity shift: over-grows the emergent-span `dmc_dma_step` reload → Explicit `04→05`,
  Implicit `00→02` regress. C1 preserved throughout (apu_phase stays START, φ2 on `master_clock`).
- `mc-r1-counter-exact` — **M-2 full counter rewrite, REFUTED BY MEASUREMENT** (Stage-A audit):
  - The 4 counters (`cpu.cycles`/`cycles_emitted`/`apu.cpu_cycle`/`master_clock`) do **not drift** — they're
    coherent. `mc-r1-internal-cycle` (every-cycle-bus-access) is **fully inert**; deleting `mc-r1-dma-cycle-budget`
    would not help. **The plan's "collapse counters → per-context exactness" mechanism does not hold.**
  - The Y=3→4 lever is **purely** the put_cycle end-flip, **inseparable** across the hardcoded-span DMC+OAM
    overlap and the emergent-span `dmc_dma_step` reload (one shared byte-timer phase).
  - All 3 surgical per-context separations on `dmc_dma_step` WEDGE (GET-on-apu_phase → Y=0; halt-clear-on-apu_phase;
    reload-halt-preclear). DO-NOT-RETRY table in `v2.0-m2-counter-rewrite-stage-a-2026-06-08.md`.
