## Where A0/A1 sit

Sprint A replaces the integer-PPU-dot-per-CPU-cycle lockstep with a `u64`
master clock + run-to-timestamp catch-up (the TetaNES-validated model), so the
CPU memory access lands at its true sub-cycle position within the cycle ‚Äî the
single axis that unblocks the C1 IRQ family, `$2002` sub-cycle timing, PAL
3.2:1, and FDS. **A0** is the no-behavior-change groundwork; **A1** lands the
master clock behind a default-off feature flag (correctness comes in A2+).

**Already done (v1.6.0 Sprint 5):** the perf trip-wire half of A0 ‚Äî
`scripts/bench_regression_check.sh` + the `bench` CI job (absolute frame-time
ceiling) and the rendering-heavy `nes_run_frame_flowing_palette` bench. A0 below
is therefore reduced to the *region-as-data* refactor.

**Already present (landed across v1.0.0 ‚Üí v1.5.0, the foundation A1 builds on):**
- `crates/nes-cpu/src/scheduler.rs` ‚Äî `M2Phase::{Low,High}` enum + `poll_irq_at_phase`.
- `crates/nes-core/src/bus.rs` ‚Äî the `cpu-c1-attempt-17-access-reorder` feature scaffold: `tick_cpu_cycle_phi1` / `tick_cpu_cycle_phi2` (‚âà bus.rs:1330+), each already creating a `PpuBusAdapter` (bus.rs:1154 / :1171) and ticking the PPU; the `irq_snapshot_*_at_{low,high}` fields; `current_m2_phase()`.
- `crates/nes-cpu/src/cpu.rs` ‚Äî `read1`/`write1` with the œÜ1/œÜ2 split path (‚âà cpu.rs:529) under the same feature.
- Mesen2 oracle tooling: `scripts/mesen2_cpu_boot_trace.lua` (per-instruction), `scripts/mesen2_irq_trace.lua` (per-IRQ-event), the Mesen2 `EventType::PpuCycle` source patch (per-PPU-cycle), `crates/nes-test-harness/golden/irq_trace/mesen2/` baselines.
- The ADR-0007 equivalence-harness pattern: `crates/nes-test-harness/tests/dmc_get_put_equivalence.rs` + `scripts/dmc_equivalence_harness.sh` (the template for A1's parallel-impl validation).

## Sprint A0 ‚Äî Region-as-data dividers (‚âà 2-3 engineer-days)

**Goal:** express the CPU:PPU clock relationship as per-region *data* (master
clocks per CPU cycle, master clocks per PPU dot), with **zero behavior change**,
so A1 can swap the scheduler without also touching region logic.

### Current state
- `crates/nes-mappers/src/cartridge.rs:60` ‚Äî `enum Region { Ntsc, Pal, Dendy }`.
- `crates/nes-ppu/src/ppu.rs:19` ‚Äî `enum PpuRegion { Ntsc, Pal, Dendy }` (per-region scanline/VBL constants live here already, landed in v1.5.0 Sprint 3).
- `crates/nes-core/src/bus.rs` ‚Äî `tick_one_cpu_cycle` ticks the PPU **3 dots** per CPU cycle in a `for sub_dot in 0..3` loop (PAL is approximated as 3, not 3.2 ‚Äî the documented v2.0 gap).

### Changes
1. **Add a region divider table** (new `RegionTiming` in `nes-core` or extend `PpuRegion`):
   - `cpu_divider`: master clocks per CPU cycle = **12** (NTSC) / **16** (PAL) / **15** (Dendy).
   - `ppu_divider`: master clocks per PPU dot = **4** (NTSC) / **5** (PAL) / **5** (Dendy).
   - `ppu_offset`: the cold-boot CPU‚ÜîPPU alignment constant (the `_ppuOffset` analog; Mesen2 `NesCpu.cpp`, TetaNES `cpu.rs` `PPU_OFFSET`). Single integer; A3 calibrates it.
   - Ratios: 12/4 = 3.0 dots/cyc (NTSC ‚úì), 16/5 = 3.2 (PAL ‚Äî the fix), 15/5 = 3.0 (Dendy). Cross-check against `nesdev_wiki` `Cycle_reference_chart`.
2. **Route the current `0..3` loop through the data** without changing output: `tick_one_cpu_cycle` reads `dots_per_cycle = cpu_divider / ppu_divider` (integer 3 for NTSC/Dendy; PAL still rounds to 3 here ‚Äî A6 fixes PAL once the master clock exists). This is a pure refactor: the emitted dot count is unchanged for every region.
3. Keep `Region` (cartridge) ‚Üí `PpuRegion`/`ApuRegion` mapping as-is; just attach the divider table.

### Exit criteria
- `cargo test --workspace --features test-roms` byte-identical: **702 strict + 10 ignored**, AccuracyCoin 90.65%, 60-ROM oracle 60/60, sacred trio + B4 invariant unchanged.
- The `bench` gate stays green (no measurable delta).
- No new feature flag; this is unconditional groundwork.

### Files
`crates/nes-mappers/src/cartridge.rs` (Region, untouched semantics), `crates/nes-ppu/src/ppu.rs` (PpuRegion + divider data), `crates/nes-core/src/bus.rs` (`tick_one_cpu_cycle` reads dividers).

## Sprint A1 ‚Äî Master-clock core behind `master-clock-scheduler` (‚âà 5-8 engineer-days)

**Goal:** introduce a `u64` master clock + `run_to(timestamp)` catch-up as the
**feature-on** scheduling path, leaving the default path byte-identical.
Correctness of the new path (flipping C1 etc.) is A2+; A1's bar is "default-off
unchanged; feature-on produces *a* frame and the equivalence harness runs."

### A1.1 ‚Äî Feature flag + master-clock state
- Add cargo feature `master-clock-scheduler` to `nes-core` (+ propagate to `nes-cpu`, `nes-ppu` as the œÜ1/œÜ2 path already does for `cpu-c1-attempt-17-access-reorder`). Default **off**.
- On `LockstepBus`: `master_clock: u64`, `cpu_divider: u8`, `ppu_offset: u8` (from A0's table). On `Ppu`: `master_clock: u64` + `ppu_divider: u8` (or keep the PPU's clock on the bus and pass a target ‚Äî decide in A1.2).
- Initialize in the constructor + `reset`/`power_cycle`; serialize in the BUS + PPU save-state sections **only under the feature** (or always, with trailing-default back-compat ‚Äî prefer the latter so feature-on/off saves inter0perate during bring-up).

### A1.2 ‚Äî `ppu_run_to(target)` catch-up (the load-bearing piece)
- New bus method: `ppu_run_to(&mut self, target_master_clock: u64)` = `while self.ppu_master_clock + ppu_divider <= target { <tick one dot>; self.ppu_master_clock += ppu_divider }`. This ticks the PPU the **same number of dots per CPU cycle** as today (‚âà3), so the `ppu_throughput` cost is unchanged ‚Äî the catch-up loop is what keeps the refactor inside the 8√ó perf headroom (it is *not* a per-master-clock uniform tick).
- **The one non-trivial refactor ‚Äî the per-dot `PpuBusAdapter` mapper borrow** (bus.rs:1154 / :1171): the adapter borrows `self.mapper.as_mut()` per dot. Move adapter creation *inside* the `ppu_run_to` loop so the mapper borrow is released between catch-up and the CPU's own mapper access. Sound because the bus owns `mapper` exclusively; if the borrow checker fights, use a scoped `self.mapper.as_mut()` re-borrow per dot (already the per-dot pattern today).

### A1.3 ‚Äî Promote œÜ1/œÜ2 to the feature-on production path
- `tick_cpu_cycle_phi1` (bus.rs:1330) ‚Üí run the PPU to `master_clock + start_clocks` (the pre-access advance, ‚âà 5/12 of the cycle per Mesen2 `NesCpu.cpp` `StartCpuCycle`/`MemoryRead`), then the access fans out.
- `tick_cpu_cycle_phi2` ‚Üí run the PPU to the cycle end (`master_clock + cpu_divider`), then end-of-cycle bookkeeping (the `notify_cpu_cycle` / `tick_with_external` / IRQ-high snapshot currently at the end of `tick_one_cpu_cycle`); advance `master_clock += cpu_divider`.
- `crates/nes-cpu/src/cpu.rs` `read1`/`write1` (‚âà cpu.rs:529): under `master-clock-scheduler`, take the œÜ1 ‚Üí access ‚Üí œÜ2 path (the scaffold already exists for `cpu-c1-attempt-17-access-reorder`; unify or supersede that flag).
- The CPU stays the active driver; `run_frame` (nes.rs:165) outer loop is unchanged.

### A1.4 ‚Äî Equivalence harness + validation
- New `crates/nes-test-harness/tests/master_clock_equivalence.rs` (mirror `dmc_get_put_equivalence.rs`): run a fixed ROM under both schedulers (feature off vs on), report a per-frame match-count signal (framebuffer + audio FNV-1a + cycle), plus a `#[ignore]`'d strict byte-equivalence probe per the project's `*_currently_fails` contract. During A1 the two paths will *differ* (that's the point ‚Äî the new path moves the access); the harness is the per-sprint regression sentinel as A2-A7 converge the new path onto Mesen2.
- Validate the feature-on path against **Mesen2** (not the old baseline): `mesen2_cpu_boot_trace.lua` (first `SEI` at the same master clock ‚Äî the `ppu_offset` calibration check), `mesen2_irq_trace.lua`, the `EventType::PpuCycle` per-PPU-cycle patch.

### Exit criteria
- **Default-off byte-identical**: 702 strict + 10 ignored, oracle 60/60, sacred trio + B4 + AccuracyCoin 90.65% unchanged; `bench` gate < 5% on `full_frame`.
- **Feature-on**: the workspace builds with `--features master-clock-scheduler`; the equivalence harness runs and emits its match-count; the emulator produces a frame (no panic, no hang). Correctness (flipping C1 / `$2002`) is explicitly **deferred to A2-A3**.

### Risks (carried from plan ¬ß6)
- **Borrow-checker** (A1.2) ‚Äî confine the `PpuBusAdapter` lifetime inside `ppu_run_to`.
- **`ppu_offset` calibration** (touched in A1, calibrated in A3) ‚Äî wrong value re-introduces the Session-13 +344-dot class of regression; detect via the per-instruction Mesen2 boot trace.
- **Perf** ‚Äî the catch-up loop must keep the dot count identical; re-profile with the `bench` gate + `perf` after A1.

### Files
`crates/nes-core/Cargo.toml` (feature), `crates/nes-core/src/bus.rs` (master-clock state, `ppu_run_to`, œÜ1/œÜ2 promotion, PpuBusAdapter borrow), `crates/nes-cpu/src/cpu.rs` (read1/write1 path), `crates/nes-cpu/src/scheduler.rs` (master-clock helpers), `crates/nes-ppu/src/ppu.rs` (ppu master_clock/divider), `crates/nes-core/src/bus_snapshot.rs` + the PPU snapshot (feature/back-compat fields), `crates/nes-test-harness/tests/master_clock_equivalence.rs` (NEW).
