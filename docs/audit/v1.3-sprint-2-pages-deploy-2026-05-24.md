# v1.3.0 Sprint 2 — GitHub Pages deploy + CI wasm32 gate + size budget

**Date:** 2026-05-24
**Status:** landed
**Scope:** ship the wasm32 frontend to GitHub Pages; gate every PR on a
wasm32 build + the wasm-specific clippy lints; enforce a compressed
size budget on the deployed bundle.

---

## What landed

Three pieces of CI/deploy infrastructure. No Rust source changed — this
sprint is entirely workflow + tooling + host-page copy.

### 1. `scripts/wasm_size_budget.sh` (NEW)

Sums the **gzip**-compressed size of the shippable `dist/` assets (the
`.wasm` module + the wasm-bindgen `.js` glue) and fails if the total
exceeds a byte budget (default 5 MiB). Prints a per-asset raw / gzip /
brotli table.

- Gates on **gzip**, not raw `.wasm` size: GitHub Pages serves these
  assets compressed over the wire, so the compressed total is what
  governs cold-cache download time.
- Brotli is printed for information (Pages' Fastly CDN prefers it) but
  the PASS/FAIL gate is gzip — universally available and strictly
  larger than brotli, hence the conservative bound.
- Runnable locally: `scripts/wasm_size_budget.sh [DIST_DIR] [BUDGET]`.

### 2. `.github/workflows/web.yml` (NEW) — Pages deploy

- **build job** (runs on push to `main`, PRs, and manual dispatch):
  installs the wasm32 target + a pinned prebuilt **trunk 0.21.14**
  (trunk itself auto-downloads the wasm-bindgen CLI + wasm-opt versions
  pinned in `Trunk.toml`, so the workflow installs nothing else), runs
  `trunk build --release --public-url /RustyNES_v2/`, enforces the size
  budget, and uploads the Pages artifact.
- **deploy job** (push / dispatch only — **never** on PRs): publishes
  via `actions/deploy-pages@v5`.
- `actions/configure-pages@v6` with `enablement: true` self-heals if
  Pages is ever disabled.
- All Pages actions are pinned to their **Node 24** lines
  (`configure-pages@v6`, `upload-pages-artifact@v5`, `deploy-pages@v5`)
  — the `@v5`/`@v4`/`@v3` predecessors ran on the deprecated Node 20
  runtime (GitHub force-migrates to Node 24 on 2026-06-02).
- **No `.nojekyll`.** The GitHub Actions Pages flow serves the uploaded
  artifact verbatim (Jekyll only runs in the legacy branch-deploy
  mode), and no trunk output filename starts with `_`, so there is
  nothing for Jekyll to strip even if it ran. `upload-pages-artifact`
  >= v4 also excludes dotfiles by default, so a `.nojekyll` would be
  dropped from the artifact regardless.
- Permissions are minimum-necessary: `pages: write` + `id-token: write`
  for the deploy, `contents: read` otherwise. `concurrency: pages`
  (no cancel) serializes deploys.
- The deployed default is the **`wasm-winit`** feature: the full
  winit + wgpu + egui experience (debugger overlay + NTSC filter),
  identical to the native binary.

### 3. `ci.yml` — new `wasm` job

`cargo clippy -p nes-frontend --target wasm32-unknown-unknown --lib
--bins -- -D warnings`, run for **both** `#[wasm_bindgen(start)]`
flavours (`wasm-winit` default + `wasm-canvas` embed).

This closes a real coverage gap: the wasm32 frontend modules
(`wasm.rs` / `wasm_winit.rs` / `wasm_audio.rs`) only compile under
`cfg(target_arch = "wasm32")`, so the existing native `clippy` job
**never lints them**. This job is the only PR-time signal for that code.
No apt deps are installed — the wasm32 build of winit/wgpu/cpal routes
through web-sys, not the native X11/wayland/ALSA system libraries.

## `public_url` strategy

`Trunk.toml` keeps `public_url = "/"` so `trunk serve` works locally
with no flags. The Pages deploy overrides it on the CLI
(`--public-url /RustyNES_v2/`) because repository ("project") Pages
serve under `/<repo>/` and every asset URL trunk emits must be
prefixed. Verified locally: `dist/index.html` references
`/RustyNES_v2/nes-frontend-<hash>.js` + `_bg.wasm`.

## Size budget — measured

Fresh `trunk build --release` (wasm-opt `-Oz`) of the default
`wasm-winit` build:

| asset | raw | gzip | brotli |
|---|---:|---:|---:|
| `nes-frontend-<hash>_bg.wasm` | 5,301,399 | 2,203,756 | 1,725,639 |
| `nes-frontend-<hash>.js` | 126,316 | 18,422 | 15,529 |
| **TOTAL** | 5,427,715 | **2,222,178** | 1,741,168 |

- gzip total **2.12 MiB** vs the **5 MiB** budget — **2.88 MiB
  headroom** (PASS).
- brotli (what the CDN actually serves) is **1.66 MiB**.

The lightweight `wasm-canvas` embed build is ~316 KB (per Sprint 1.4)
and trivially under budget; the gate measures the heavier default.

## Operational notes

- **First deploy enables Pages.** `configure-pages` with
  `enablement: true` provisions the site on the first `main` push. If
  the repo's org/account policy blocks Actions from enabling Pages, a
  one-time manual flip of *Settings → Pages → Source: GitHub Actions*
  is required; subsequent deploys are automatic.
- **Live URL:** `https://doublegate.github.io/RustyNES_v2/`.
- **Bumping trunk:** change `TRUNK_VERSION` in `web.yml` and the
  `wasm_bindgen` pin in `Trunk.toml` together (the CLI + library
  version-check each other).

## Not in scope (deferred)

- COOP/COEP headers for `SharedArrayBuffer` / AudioWorklet —
  `ScriptProcessorNode` audio (Sprint 1.4c) needs neither, and GitHub
  Pages cannot set response headers anyway.
- A deploy preview per-PR (would need a separate environment); PRs
  build + size-gate but do not publish.
- Cross-browser smoke automation (Playwright). The v1.3.0 exit
  criterion of "loads in Chrome / Firefox / Safari" stays a manual
  check, recorded in the release notes.
