# RustyNES v2.0.1 — Single-Path Cleanup

**Release date:** 2026-06-10
**Type:** maintenance / refactor (no behaviour change on the shipping build)

## TL;DR

| | |
|---|---|
| **What changed** | The legacy integer-lockstep scheduler was **removed entirely** — the R1 master clock is now the only path. |
| **Accuracy** | **AccuracyCoin 100.00% (139/139)** — unchanged. |
| **Behaviour** | **Byte-identical** on the shipping build (60-ROM commercial oracle 60/60, nestest 0-diff, visual_regression 7/7, save-state determinism). |
| **Code** | **−3334 / +57** lines across 72 files; 45 files deleted. |
| **Breaking** | Feature flags only — no API change. Embedders enabling the removed cargo features must drop them. |

## Why

v2.0.0 made the R1 master-clock scheduler the default (AccuracyCoin 100%) but kept
the v1.7.0 integer-lockstep scheduler reachable via `--no-default-features`
(94.24%) so the two could be A/B-compared. That comparison did its job — it
confirmed R1 was worth keeping as the sole scheduler (a 6–8% frame-time cost for
the +5.76-point accuracy gain, with the cost isolated to bus-side accuracy work;
see [`docs/benchmarks.md`](../benchmarks.md)). With that settled, the legacy path is
now dead weight: a second scheduler, a second DMA model, and ~30 feature gates
threaded through four crates. v2.0.1 removes it.

## What was removed

- **The entire legacy integer-lockstep scheduler**: the per-dot lockstep CPU loop,
  the two-driver DMA model (`dmc_overlap_*`, the legacy `service_dmc_dma*` /
  `oam_dma_step` / `clock_oam_dma_cycle` drivers), and the cycle-START DMC
  clock/arm path.
- **28 now-permanent R1 feature flags, unconditionalised** — the `mc-r1-full-cpu`
  umbrella and its entire closure (`mc-r1-substrate`, the unified-DMA engine chain
  `mc-r1-dma-unified` → `-dma-unified-collapse` → `-counter-collapse` →
  `-dmc-bytetimer-end` → `-dmc-oam-overlap` → `-dmc-reload-visibility-delay`,
  `mc-r1-dmc-delayed-4015`, `-dmc-abort-cancel`, `-dmc-abort-timer-phase`,
  `-dmc-reenable-phase`, `-dmc-halt-subpos`, `-dmc-load-get-entry`,
  `-dmc-idle-halt`, `-brk-padding-read`, `-branch-poll-points`,
  `-apu-unified-clock`, `-one-clock`, `-oam-dma-reg-window`, `mc-ppu-subpos`,
  `mc-ppu-2007-render-buffer`, `cpu-stack-dummy-reads`, `cpu-implied-dummy-reads`)
  plus the PPU-accuracy trio (`ppu-oam-data-bus`, `ppu-sprite-shifter-counter`,
  `ppu-2002-read-end-flags`) and `accuracycoin-sprite-eval-base-from-oamaddr`. The
  code they gated is now the only code.
- **4 dead/superseded experiment flags**: `cpu-c1-attempt-17-access-reorder`
  (rolled-back C1 attempt), `mc-r1-coldboot-ppuoffset` + `mc-r1-bp-ppu-offset`
  (refuted offset sweeps), and `dmc-get-put-scheduler` (ADR-0007, superseded by the
  R1 unified DMA engine) — with its `dmc_get_put_equivalence.rs` test and two
  diagnostic sweep bins.
- **Legacy-only tests/artifacts**: the bbbradsmith `audio_tests` framebuffer corpus
  (it asserted the pre-R1 DMC audio hashes; R1 changed DMC timing by design) + its
  19 snapshots + 22 screenshots, and two dead diagnostic scripts.

## What was kept

All real and dev/diagnostic feature flags are untouched: `std`, `serde`,
`test-roms`, `commercial-roms`, `mapper-audio`, `wasm-winit` / `wasm-canvas`, and
the tracing/diagnostic flags `irq-timing-trace`, `ppu-state-trace`,
`cpu-boot-trace`, `cpu-instr-cycle-trace`, `mc-r1-dmc-abort-probe`.

Default features now: `rustynes-core` → `["std"]`, `rustynes-frontend` → `["wasm-winit"]`,
`rustynes-test-harness` → `[]`. Run the test suite with `--features test-roms` (drop the
now-removed `mc-r1-full-cpu`).

## Verification

This was a behaviour-preserving refactor, so the gate was **byte-identical
output**, not merely "tests pass":

- **AccuracyCoin: 100.00% (139/139, "no failing tests")**, RAM-direct decoder.
- **60-ROM commercial oracle: 60/60** — framebuffer + audio + cycle hashes all
  byte-identical to v2.0.0 (the precise detector for any mis-resolved `cfg`).
- nestest 0-diff; `cpu_interrupts_v2` 5/5 strict; SH\* 6/6; `ppu_vbl_nmi` 10/10;
  `visual_regression` 7/7; `region_timing` 4/4 (PAL 3.2:1); `save_state`
  round-trips.
- `cargo fmt --all --check`, `cargo clippy --workspace --all-targets -- -D warnings`,
  `cargo doc --workspace --no-deps` — clean. `no_std` cross-compile
  (`thumbv7em-none-eabihf`) green.

## Breaking changes

Feature flags only — there is no API change. Embedders of the `nes-*` crates that
explicitly enabled any of the removed cargo features (most notably
`mc-r1-full-cpu`, or that selected `--no-default-features` to get the legacy
scheduler) must drop those flags: the R1 behaviour they named is now
unconditional. `--no-default-features` no longer selects a second scheduler (it
only toggles `std` / the wasm flavour as before).
