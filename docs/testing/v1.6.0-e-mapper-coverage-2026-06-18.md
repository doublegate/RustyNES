# v1.6.0 "Studio" Workstream-E Mapper ROM + Screenshot Coverage

**Audit date.** 2026-06-18.
**Branch.** `feat/v1.6.0-e-coverage` (off `feat/v1.6.0-studio-completion`).
**Scope.** The 23 new v1.6.0 "Studio" Workstream-E mapper families implemented
in `crates/rustynes-mappers/src/sprint11.rs` (the `126 -> 150 BestEffort`
sweep, commit `deda1d4`). These were implemented + register/save-state
unit-tested but had **no staged commercial ROMs and no screenshots** — the
earlier coverage bless only captured already-staged ROMs, so this batch was a
pure coverage gap. This pass closes it.

## What this is (and is not)

All 23 families are **Tier-2 BestEffort** (`crates/rustynes-mappers/src/tier.rs`,
ADR 0011): never accuracy-gated, never backing a ROM in the AccuracyCoin /
oracle corpus. This work is **capture-only** — it stages developer-local
commercial dumps (gitignored under `tests/roms/external/`, never committed) and
commits only the emulator's *output*: one screenshot per booting game (under
`screenshots/besteffort/mapper-NNN-mNN/`) and the machine-readable boot-smoke
baselines (`crates/rustynes-test-harness/tests/snapshots/external_coverage__*.snap`).
No core/code change, so the shipped / native / `no_std` / wasm builds stay
byte-identical and **AccuracyCoin holds 100% (139/139)**.

Pipeline used (the existing one — `scripts/coverage/`):
`index -> discover -> stage --execute -> bless.sh -> visual curate -> cargo insta accept`.
Library root: `~/Dropbox/ROMs/Nintendo Entertainment System - Famicom (2020)/`
(24,780 iNES headers indexed). Screenshots are the f1100 boot-smoke checkpoint
unless noted. Captures that came up blank / solid / glitched / dithered are
**not** committed (a misleading screenshot is worse than none) and are recorded
below as broken-boot residuals — these all converge on later mapper-accuracy
work, not this coverage pass.

### Summary

- **13 families with at least one good committed screenshot** (19 screenshots
  total): 42, 44, 46, 49, 52, 57, 115, 120, 134, 138, 139, 141, 189.
- **4 families staged but broken-boot** (no committed screenshot — blank / solid
  / dither): 50, 51, 205, 245, 290.
- **5 families = known gaps** (no usable dump): 104, 238, 301, 348, 366.

## Known gaps + residuals (do not fabricate coverage)

- **104 (Golden Five), 238, 366 (BMC-GN-45)** — no iNES *or* UNIF dump exists in
  the indexed library. No screenshot possible until a dump is sourced.
- **301 (BMC-8157), 348 (BMC-830118C)** — a `.unf` (UNIF) dump exists in the
  library (`4-in-1 1993 (CK-001)`, `7-in-1 (M-022)`), but the UNIF cartridge
  loader's board table (`crates/rustynes-mappers/src/unif.rs`) does not yet map
  those two board strings to mappers 301/348, so the dumps cannot currently be
  loaded. Adding those two board-name entries is a small follow-up (out of scope
  for this capture-only pass, which touches no code). Once mapped, re-stage
  with `coverage.py stage --unif --execute` and re-bless.
- **50, 51, 205, 245, 290** — dumps boot to a blank / solid / dithered frame.
  These are BestEffort register-decode ports; a meaningful frame likely needs
  finer banking / IRQ-timing fidelity (a later mapper-accuracy pass), not more
  ROMs. Their boot-smoke `.snap` baselines are still committed to pin current
  behavior and catch any future regression/improvement.
