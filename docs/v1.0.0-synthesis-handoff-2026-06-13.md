# RustyNES v1.0.0 synthesis — handoff

**Date:** 2026-06-13
**Audience:** the next Claude Code (or human) working in this repo.
**TL;DR:** The original RustyNES (v0.8.6) emulation core was **replaced wholesale**
with the cycle-accurate **RustyNES v2 (engine) v2.8.0** technology, renamed into
this repo's `rustynes-*` crate namespace, the original desktop UX polish was ported

If you last edited this repo at v0.8.x, **most of your assumptions are now obsolete.**

## 1. What happened

Two sibling projects existed:

- **The engine** (`Commercial_Private-Projects/RustyNES_v2`, was v2.8.0) — a
  cycle-accurate NES emulator: AccuracyCoin 100% (139/139), nestest 0-diff, the
  Mesen2/higan/ares accuracy bar (PPU-dot lockstep scheduler). 51 mappers, FDS,
  Vs/PC10, rollback + WebRTC netplay, RetroAchievements, TAS movies, save-states,
  rewind, run-ahead, display-sync pacing, a dedicated emulation thread, an egui
  debugger, and a WASM/GitHub-Pages build.
- **This repo** (`OSS_Public-Projects/RustyNES`, was v0.8.6) — an earlier,
  less-accurate emulator whose unique value was a **polished desktop UX** plus a
  large doc/branding ecosystem.

The engine's emulation technology was harvested into this repo, the old core was
retired, the UX was ported onto the new stack, and the docs/branding were
synthesized. **The engine project (`RustyNES_v2`) was treated strictly read-only**
throughout — every operation read from a staging copy placed inside this repo.

## 2. The architecture now (v1.0.0)

A pure-Rust Cargo workspace. Crates (all `rustynes-*`):

| Crate | Role |
|---|---|
| `rustynes-cpu` | 2A03 CPU (6502 core) |
| `rustynes-ppu` | 2C02 PPU |
| `rustynes-apu` | 2A03 APU (band-limited synthesis) |
| `rustynes-mappers` | cartridge parsing + mapper trait + 51 mapper families |
| `rustynes-core` | the `Nes` facade, the lockstep scheduler, the Bus, save-state — re-exports the chip crates |
| `rustynes-frontend` | the desktop + wasm binary (`rustynes`); winit + wgpu + egui + cpal |
| `rustynes-netplay` | GGPO-style rollback netcode (UDP + WebRTC) |
| `rustynes-cheevos` | RetroAchievements (opt-in, native-only; vendored rcheevos C via FFI) |
| `rustynes-test-harness` | test-ROM runner, oracle, golden compares |

**Load-bearing architecture facts** (unchanged from the engine; read `docs/architecture.md` +
`docs/scheduler.md` + `CLAUDE.md`):

- **The PPU is the master clock.** The scheduler advances one PPU dot per tick; the
  CPU advances on every 3rd dot (NTSC/Dendy; 3.2nd PAL); the APU every other CPU
  cycle. Lockstep, not catch-up — this is why mid-instruction PPU events work.
- **The Bus owns everything mutable** (PPU, APU, mapper-via-cart, WRAM, controllers,
  open-bus latch). The CPU borrows `&mut Bus` during `tick()`.
- **Workspace dep graph is one-directional**: `rustynes-cpu` has no PPU/APU dep;
  `rustynes-ppu` depends on `rustynes-mappers` only; `rustynes-core` ties them
  together. Don't add a cross-chip dependency.
- **Mapper IRQ logic lives in the mapper**, not the PPU.
- **Determinism is a hard contract**: same seed + ROM + inputs ⇒ bit-identical
  framebuffer + audio. Don't introduce hidden non-determinism into the core.
- **Test ROMs are the spec.** AccuracyCoin 100% + the commercial-ROM oracle
  byte-identity is the accuracy gate.

The **frontend** is the engine's winit/wgpu/egui/cpal app PLUS the original RustyNES
desktop UX shell ported on top (Phase B): a persistent menu bar, recent-ROMs MRU, a
first-run welcome modal, a keyboard-shortcuts window, a tabbed Settings window,
light/dark/system themes, an 8:7 pixel-aspect toggle, and a status bar — all in
`crates/rustynes-frontend/src/ui_shell.rs` + the always-on egui pass in
`debugger/mod.rs::render_shell`. The binary is `rustynes`.

## 3. What was renamed and why

The engine used `nes-*` crate names and a `rustynes-v2` binary. To live in this
repo's namespace, **everything was renamed `nes-* → rustynes-*`** (crate names,
workspace members, inter-crate deps, every `use nes_core::` → `use rustynes_core::`),
the binary `rustynes-v2 → rustynes`, and `RustyNES_v2 → RustyNES`. This was a pure
mechanical rename — the emulation is **byte-identical** to the engine's v2.8.0 (proven
by AccuracyCoin 100% + the oracles staying byte-identical post-rename). Watch for the
two false-positive classes the rename hit (now fixed): a branding string
(`"rustynes-netplay"`) and a ROM-path/test name (`blargg_nes_cpu_test5`).

## 5. Hard constraints (carried over — DO NOT violate)

- **NEVER commit commercial Nintendo ROMs.** The oracle uses the **gitignored**
  `tests/roms/external/` (user supplies dumps); only CC0/public-domain test ROMs are
  committed. The large `nes-test-roms/` clone is gitignored except the ~89 wired-test
  ROMs force-committed for CI.
- **The accuracy gate** is AccuracyCoin 100% (139/139) + the 60-ROM + 52-entry
  commercial oracles byte-identical. Any core change must preserve these.
- **Determinism contract** (see §2).
- `nesdev_wiki/` (208M), `perf-logs/`, `web/dist/`, `__pycache__/` stay gitignored.

## 6. Version + build

- **Current version: v1.0.0.** (The engine's prior v1.x/v2.x line is documented as the
  **upstream engine lineage** in `CHANGELOG.md` / `docs/STATUS.md`, mapped onto the
  documentary `v0.9.0–v0.9.7` stages — those are NOT separate releases of this repo.)
- Build/test (rustynes-* crate names; edition 2021, toolchain 1.86):
  - `cargo build --workspace`
  - `cargo test --workspace --features test-roms` (AccuracyCoin + ROM suites)
  - `cargo run --release -p rustynes-frontend -- path/to/rom.nes`
  - Gates: `cargo fmt --all --check`; `cargo clippy --workspace --all-targets -- -D warnings`;
    `RUSTDOCFLAGS=-D warnings cargo doc --workspace --no-deps`;
    `cargo build -p rustynes-core --target thumbv7em-none-eabihf --no-default-features`;
    wasm via `trunk` (`crates/rustynes-frontend/web`).
  - **`cargo doc` runs with `RUSTDOCFLAGS=-D warnings` in CI** — a broken intra-doc
    link fails CI even though a bare local `cargo doc` passes. Always run the
    `-D warnings` form before pushing.
- CI/CD: `.github/workflows/ci.yml` (lint + OS-matrix test + test-roms + no_std + wasm +
  bench), `release.yml` (tag-triggered binaries; never writes the release body —
  anti-clobber), `web.yml` = **"Deploy Pages (demo + docs)"** (builds the wasm demo AND
  the workspace rustdoc and publishes BOTH in one deployment), `security.yml`
  (cargo-audit/deny), + the shared `actions/rust-setup` composite.
- **GitHub Pages serves two things from one deployment**: the playable wasm demo at the
  site **root** (`/`) and the rustdoc at **`/api/`** (with a redirect to `rustynes_core`).
  Pages is on the **"GitHub Actions" source** (`build_type: workflow`), NOT a branch —
  the old `gh-pages` branch + the GitHub-managed `pages-build-deployment` dynamic workflow
  are decommissioned (the branch is deleted; that built-in workflow can't be removed but is
  inert). Don't re-add a branch-based docs deploy; extend `web.yml` instead.
- **wasm-bindgen pin gotcha**: `crates/rustynes-frontend/web/Trunk.toml` pins the
  wasm-bindgen **CLI** version, which MUST exactly equal the wasm-bindgen **library**
  version in `Cargo.lock` (`grep -A1 'name = "wasm-bindgen"' Cargo.lock`) — a mismatch
  fails `trunk build` (and the Pages deploy) at the wasm-bindgen step, but wasm *clippy*
  passes, so it's invisible unless you run the full `trunk build`. Bump the pin whenever a
  resolve moves the library version.
- **RetroAchievements client identity**: the RA HTTP User-Agent (the string RA
  authenticates/identifies/allowlists the client by) is `RustyNES/<crate version>
  rcheevos/<rcheevos version>` (e.g. `RustyNES/1.0.0 rcheevos/12.3.0`) — defined as
  `RA_USER_AGENT` in `crates/rustynes-cheevos/src/http.rs`; the rcheevos version is parsed
  from the vendored `rc_version.h` by `build.rs` (`RCHEEVOS_VERSION`), so it auto-syncs on
  a re-vendor. Keep the leading token `RustyNES/...` (a regression test guards it).

- **Bug-fix rounds 1–3** — menu submenu hover-close; 8:7 / fullscreen letterboxing;

  AccuracyCoin 100% + the oracles stay byte-identical): master volume + mute,
  per-APU-channel mutes, emulation-speed presets (25–300 %, keys `=`/`-`/`0`, with
  pitch-shifted audio), a thumbnail save-state manager, overscan crop, pause-dim,
  a gamepad deadzone slider, controller hot-plug toasts, screenshot-to-clipboard, a
  per-section Reset-to-Defaults, and a Performance-panel frame-time graph. See the
  `CHANGELOG.md` `[1.0.0]` entry.
- An `[resolver] incompatible-rust-versions = "fallback"` guard in `.cargo/config.toml`
  keeps `arboard`'s transitive `image` dep from drifting past the 1.86 MSRV on a future
  `cargo update`.

## 9. The v1.0.0 release — shipped & live

v1.0.0 was cut and published on 2026-06-13:

- **Tag + GitHub release**: `v1.0.0` on `main`, with Linux / macOS (aarch64) / Windows
  binaries attached by `release.yml`. Intel macOS was intentionally dropped (ADR 0009).
- **`main` history**: `feat/v1.0.0-synthesis` FF-merged to `main`, then a stray Dependabot

  the old root `Cargo.toml`, the old `release.yml`), so the tested tree was kept verbatim.
- **Pages live**: demo at <https://doublegate.github.io/RustyNES/>, docs at `/api/` (see §6).
- **Post-release polish on `main`** (all green; AccuracyCoin 100% + oracles byte-identical
  throughout): the #4 feature batch (§8); the RA User-Agent fix (§6); a scrub of stray
  "RustyNES v2" / `RustyNES_v2` leftovers the re-versioning missed (the demo page title + its
  wrong-repo link, the native window title / CLI `--help` / wasm console logs, the
  `rustynes-cheevos` `repository` metadata); removed a committed `salvaged/` cruft dir; the
  combined-Pages + wasm-bindgen-pin fixes. Deliberately retained engine-lineage mentions
  (CHANGELOG, release-notes, `ref-docs`, this handoff, `docs/archive`, dev tooling, golden
  data) are history, not release claims — leave them.
- **Actions hygiene**: the failed/cancelled run history was deleted; only green runs remain.
