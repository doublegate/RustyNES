1//! `RustyNES` Core - NES emulator core integration layer
2//!
3//! This crate provides the main emulator integration, connecting the CPU, PPU, APU,
4//! and mappers into a cohesive NES system.
5//!
6//! # Features
7//!
8//! - **Bus System**: Memory routing between CPU and all components
9//! - **Console Coordinator**: Main emulation loop with accurate timing
10//! - **Input Handling**: Controller shift register protocol
11//! - **ROM Loading**: iNES/NES 2.0 format support
12//! - **Save States**: Deterministic serialization and compression
13//! - **Zero Unsafe Code**: All implementations use safe Rust
14//!
15//! # Example
16//!
17//! ```no_run
18//! use rustynes_core::Console;
19//! use std::fs;
20//!
21//! # fn main() -> Result<(), Box<dyn std::error::Error>> {
22//! // Load ROM file
23//! let rom_data = fs::read("game.nes")?;
24//! let mut console = Console::from_rom_bytes(&rom_data)?;
25//!
26//! // Main emulation loop
27//! loop {
28//!     // Step one frame (60 FPS)
29//!     console.step_frame();
30//!
31//!     // Get framebuffer for rendering
32//!     let framebuffer = console.framebuffer();
33//!     // ... render to screen ...
34//!
35//!     // Handle input
36//!     console.set_button_1(rustynes_core::Button::A, true);
37//! }
38//! # }
39//! ```
40//!
41//! # Architecture
42//!
43//! The core consists of several key components:
44//!
45//! - **Bus**: Routes CPU memory access to RAM, PPU, APU, and cartridge
46//! - **Console**: Orchestrates all components with cycle-accurate timing
47//! - **Input**: Emulates NES controller shift register protocol
48//! - **Save State**: Serializes/deserializes complete system state
49//!
50//! # Timing Model
51//!
52//! - Master clock: 21.477272 MHz (NTSC)
53//! - CPU: 1.789773 MHz (master รท 12)
54//! - PPU: 5.369318 MHz (master รท 4), 3 dots per CPU cycle
55//! - APU: Clocked every CPU cycle
56//! - Frame: 29,780 CPU cycles, 89,341 PPU dots
