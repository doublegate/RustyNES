## Context — what v1.5.0 leaves, and the honest scope read

By the end of **v1.5.0 "Lens"** RustyNES has: the GeraNES-class debugger
*visualization* (PPU event heatmap, input miniatures, scanline trace, NSF scope,
deep mapper panel), a Lua dev/TAS API, the `.rnm` TAS movie + Replay/TAS window
(device topology, timebase, deterministic seek), per-channel audio, the native-UI
overhaul + in-app Documentation, accessibility, and a frontend pacing/audio perf pass.

**The three research passes converge on three things:**

1. **The v2.0 fractional-master-clock refactor is NOT v1.6.0.** The remaining hard
   residuals (`mmc3_test_2/4` #3, two `apu_reset`, the `$2002` NMI-suppression race,
   PAL 3.2:1) all unify under switching the timebase from "3 dots / CPU cycle" to an
   **integer master clock** (ares: NTSC 12:4, PAL 16:5 — *exact*, not fractional;
   TriCNES: 24/12/4). It's XL + HIGH-risk (ADR 0002 stop-condition, 15+ rollbacks)
   for ~3 invisible test rows + zero production-ROM impact. It genuinely deserves to
   *be* **v2.0**. Keep deferring; v1.6.0 may *seed* it cheaply (ring-fenced TriCNES
   advance-mid-read / Observe patterns) but must not open it.
2. **"Accuracy is exhausted below v2.0" was too pessimistic.** ares achieves its
   precision on an *integer* master clock, and several behaviors RustyNES filed under
   "needs v2.0" are actually **dot/CPU-cycle-granular and adoptable now** — a real
   accuracy workstream exists without the refactor (Workstream D).
3. **RustyNES is unusually well-positioned for a TAStudio-class editor** — the
   determinism contract + existing save-state/rewind/run-ahead already provide ~80%
   of the "greenzone" + seek primitives a piano-roll editor needs.

So v1.6.0 is a **creator-power + accuracy-polish + reach** release, NOT the refactor.
The two strongest identities (the maintainer picks the weighting — decision #2):
**(A) "Studio"** — TAS authoring + debugger depth + Lua/interop (the headline,
leveraging RustyNES's unique determinism strength); **(B) "Breadth"** — mappers +
FDS-proper + A/V recording + shaders + HD-pack parity. The recommended core is the
Studio spine (A–D) with breadth (E–I) as the maintainer-selected remainder.

## Hard contract (unchanged)
Determinism: same seed + ROM + input ⇒ bit-identical FB + audio; **AccuracyCoin
139/139 + oracle byte-identical** after every change (accuracy work in D pins the
failing test ROM FIRST, then implements). New features additive / off-by-default
(shipped/wasm/`no_std` byte-identical where off); dev tooling behind
`debug-hooks`/`scripting`. Chip stack `#![no_std]` + alloc; only `rustynes-frontend`
+ `rustynes-cheevos` carry `unsafe`. Core test harness never consults a frontend DB.

## STANDING RESEARCH DIRECTIVE (carry through execution)
Research is **continual**: pin behavior against `nesdev_wiki/` + committed test ROMs;
consult **BizHawk/FCEUX** (TAS/Lua), **Mesen2** (debugger/HD-pack), **ares/higan**
(accuracy), **TriCNES/puNES** (timebase/breadth) in `ref-proj/`; and **fold in
beneficial enhancements + determinism-safe perf wins opportunistically** even when
not listed here, respecting the hard contract (measured for perf, test-pinned for
accuracy). Surface anything larger-than-a-quick-win as its own ticket.

## Workstreams

### A — TAStudio-class piano-roll TAS editor (the flagship) → beta.1
Ref: BizHawk `TAStudio` + FCEUX `TASEditor`. A thin view over three decoupled
structures — an **input log**, a density-tiered savestate **greenzone**, and a
**lag log** — where editing frame N just calls `InvalidateAfter(N)` and the
greenzone rebuilds by re-emulation. RustyNES already has the deterministic replay +
save-state/rewind/run-ahead primitives.
- **A1.** Tiered state-history manager (greenzone) with eviction + the
  `InvalidateAfter(frame)` re-emulation plumbing (reuse `Movie`/`MovieUi` + snapshot).
- **A2.** Piano-roll input-log grid: per-frame editable button cells, drag-paint,
  selection, insert/delete/clone frames, marker rows, lag-frame shading.
- **A3.** A tiny **output-only** core flag — "was a controller port polled this
  frame" — for the lag log (the only core surface; `debug-hooks`-gated, byte-identical).
- **A4 (stage 2 / stretch):** branches/multitrack, project files (`.rnmproj`),
  TASVideos-style markers/notes.
Frontend + deterministic-replay reuse; replay stays bit-identical.

### B — Movie interop + Lua driving primitives → beta.1/2
- **B1.** `.fm2` (FCEUX) + `.bk2` (BizHawk) **import/export** ↔ `.rnm` for TASVideos
  round-trip. **GOTCHA (design for it):** imported movies carry no RustyNES power-on
  seed → pin a **canonical movie-import power-on alignment** or imports desync.
- **B2.** Lua **driving** primitives: `emu.frameadvance()` + coroutine-yield (unblocks
  the bot/TAS-script ecosystem) — RustyNES's Lua is already rich (draw/setInput/
  savestate/hooks); the gaps are driving + data breadth.
- **B3.** Lua data breadth: memory **domains** + sized reads, a `movie.*` namespace,
  `joypad.get/set`. All determinism-safe (gated like `emu.write`).

### C — Debugger depth (Mesen2-class) → beta.2
v1.5.0 finished *visualization*; this is *depth*. Behind `debug-hooks`.
- **C1. Expression/conditional breakpoints + read/write/exec watchpoints** — the
  keystone (Mesen `ExpressionEvaluator`); unlocks the watch window + conditional trace
  nearly for free.
- **C2. Full hex editor** — in-place poke + freeze (route through the existing raw-cheat
  overlay, like both refs) + access-type heatmap coloring + find; OAM/PPU/APU domains.
- **C3. RAM Watch + RAM Search upgrade** — operator × compare-to matrix, sizes,
  freeze, `.wch` lists (extends Memory Compare).
- **C4. Watch window + trace format-strings/conditional logging** (free riders on C1).

### D — Off-axis accuracy cluster (NO v2.0 axis) → beta.2/3
The honest accuracy work that does NOT need the master-clock refactor (ares/higan +
nesdev). **Pin the failing test ROM first.** Scope-check against v1.4.0 (which already
verified the DMC-DMA↔controller-read conflict — confirm what's *left*).
- **D1.** Full DMC/OAM-DMA model incl. `$4016`/`$4017` **controller-read corruption**
  (higan lacks it; the biggest adoptable accuracy jump) — verify the v1.4.0 subset vs
  the complete model.
- **D2.** `$2007` 6-dot blocking-read window during rendering.
- **D3.** Buggy **sprite-overflow** evaluation; **MDR/open-bus decay** timing.
- *(Explicitly v2.0, NOT here: `$2002` NMI-suppression race, `apu_reset` `$4017`
  half-cycle, `mmc3` #3, PAL 3.2:1, sprite-0 stale-shifter — the last is lowest-value;
  ares doesn't even model it.)*

### E — Mapper-breadth continuation → beta.3 (stretch)
Honesty-gated BestEffort **123 → ~150**. puNES enumerates ~370; porting the
**JYASIC** + **OneBus** ASIC cores once cascades many pirate/multicart families.
Register-decode + save-state tested; never accuracy-gated.

### E2 — UNIF (`.unf`) cartridge loader → beta.3 (maintainer-requested)
RustyNES currently rejects UNIF (`docs/cartridge-format.md` §"out of v1.0 scope").
Add a UNIF loader: parse the 32-byte header + the `MAPR`/`PRG?`/`CHR?`/`MIRR`/
`BATR`/`NAME` chunks, map the `MAPR` **board name → iNES mapper number** via a
board table (nesdev UNIF list), and route into the existing `Mapper` dispatch.
This unlocks the pirate/multicart **UNIF-only** dumps in the library that have no
iNES equivalent — e.g. `BMC-NTD-03` (**m174**) and the family of multicart boards —
so they become bootable + screenshottable. **Coverage motivation:** of the post-
v1.5.0 mapper-coverage gap, several families have ROMs ONLY as UNIF (the library
holds **90 distinct UNIF boards / 222 `.unf` files** under the nested folders);
those UNIFs are being **pre-staged now** into their per-mapper dirs (gitignored)
so screenshots can be captured the moment this loader lands. Determinism/oracle
unaffected (additive format path); honesty gate still applies to the resolved tier.

### F — FDS-proper → beta.3/4 (core-touching, NOT v2.0)
puNES `fds.c`: `$4032` auto-insert + per-game CRC quirk table + timed disk position.
Closes the Kid Icarus side-B post-registration deferral. Cycle-count-based (not the
master-clock axis); re-verify FDS determinism + the real-BIOS boot suite.

### G — A/V recording → beta.4 (RustyNES has none)
Video (frame/lossless or ffmpeg-pipe) + synchronized audio capture, off the
deterministic output. Fold in the FCEUX-style **Code/Data Logger** (output-only
PRG/CHR coverage side-array behind `debug-hooks`) here or in C.

### H — HD-pack parity continuation → beta.4 (stretch)
Mesen2 gap vs ADR 0014: **HD audio** (`<bgm>`/`<sfx>` + the `$4100` device + OGG
mixer — the biggest gap), spatial conditions (`tileNearby`/`spriteNearby`/
`positionCheck`), `<addition>`/`<fallback>`/`<options>`, full blend/priority/parallax
compositor. Behind `hd-pack`, output-only.

### I — Shader/filter ecosystem → beta.4 (stretch)
RetroArch `.slangp`/`.cgp` import (v1.4.0 rejected `.slangp` — reconsider with a
constrained translator), LMP88959 NTSC/PAL, hqNx/xBRZ. Extends the v1.2.0 ShaderStack.

*Netplay (from the Mesen2 pass — opportunistic, not a workstream): spectator mode,
adaptive input buffer, auto config-resync, a HistoryViewer timeline scrubber, rewind
XOR-delta+zlib compression. RustyNES's rollback already exceeds Mesen2's lockstep.*

## Verification (every beta + finals)
`cargo test --workspace --features test-roms` → **AccuracyCoin 139/139** + oracle
byte-identical + nestest 0-diff; `fmt --check`; `clippy -D warnings` per-feature
(default + `scripting,hd-pack` + `retroachievements` + both wasm — never
`--all-features`); `RUSTDOCFLAGS=-D warnings cargo doc`; `no_std` cross (`thumbv7em`);
markdownlint v0.39.0; wasm `trunk` + size budget; `full_frame` bench if any hot-path
touched. Concurrency/replay tests must be **deterministic** (no fixed-poll races — see
the v1.5.0 H1 present-buffer-test fix).

## Out of scope / deferred to v2.0 (document, don't build)
The **integer master-clock refactor** (the `$2002` race / `apu_reset` / `mmc3` #3 /
PAL 3.2:1 residuals — ADR 0002); **full Vs. DualSystem dual-core** (no committable
oracle); **mobile** (iOS/Android); the sprite-0 stale-shifter (lowest-value; ares
omits it). v1.6.0 may seed v2.0 with ring-fenced TriCNES advance-mid-read / Observe
patterns, but does not open the refactor.
