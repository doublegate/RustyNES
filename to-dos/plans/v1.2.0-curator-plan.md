# RustyNES — Plan: v1.2.0 "Curator"

## Context

RustyNES shipped **v1.1.0 "Scriptable"** (tag `v1.1.0`, commit `75e42cd`, 2026-06-15) on top of
the cycle-accurate **v1.0.0** production core: AccuracyCoin **100% (139/139)**, 51 mapper families,
all expansion audio (VRC6/VRC7-OPLL/Sunsoft-5B/Namco-163/MMC5/FDS — VRC7 FM confirmed *landed* via
the emu2413 port, ADR 0006), FDS, Vs./PC10, rollback netplay (UDP + WebRTC), RetroAchievements
(native), TAS, save-states/rewind/run-ahead, the flagship native Lua engine, NSF player, NES_NTSC/CRT
filters, and a WASM/Pages build. Performance has large headroom (~24% of the NTSC frame budget; a
complete PGO recipe exists but is not gated in CI).

**v1.2.0 is the next feature release.** It is deliberately broad — closing the documented native
follow-ups, pushing toward the web, widening compatibility, and mining the newly-added **GeraNES**
reference (258 mappers, an in-app per-game ROM-DB editor, a stackable shader pipeline, HD-pack/mod
loading, deep peripheral support) for the best ideas. Theme: **library breadth + compatibility +
reach** — getting more games to *load*, *run correctly*, and *be playable anywhere*, plus the polish
to curate them. (Codename "Curator" is a suggestion; rename freely.)

**Hard contract (unchanged from v1.1.0, applies to everything below):**
- Determinism: same seed + ROM + input ⇒ **bit-identical** framebuffer + audio. **AccuracyCoin
  100% (139/139)** and the commercial-ROM oracles **byte-identical** must hold after every change.
- The chip stack stays `#![no_std]` + alloc; only `rustynes-frontend` + `rustynes-cheevos` carry
  `unsafe`. Hot paths: no allocation, fixed arrays, profile before abstracting; ≤ 2 ms/frame headless.
- New features are **off by default** so the shipped, wasm, and `no_std` builds stay byte-identical
  to v1.1.0. New state-mutating paths (Lua `setInput`/writes, new cheats) are **gated off** in
  netplay / TAS replay / RA-hardcore, exactly like the existing cheat path.
- The core test harness must **never** consult any frontend database/override. Accuracy work pins a
  failing test-ROM expectation **first**, then implements until it passes (test-ROM-is-spec).

## Workstreams

The release is organized as v1.1.0 was (lettered workstreams, each independently shippable as a beta).

### A — Compatibility & mapper expansion  → beta.1

**A1. Two-tier mapper expansion** (maintainer wants *both* a curated batch and an aggressive sweep).
Introduce a `const fn mapper_tier(id, sub) -> MapperTier { Core, Curated, BestEffort }` classifier
(`crates/rustynes-mappers/src/lib.rs` or a new `tier.rs`; `no_std`, no alloc):
- **Tier 0 (existing 51):** spec-implemented, oracle/AccuracyCoin-gated. Unchanged.
- **Tier 1 (Curated, ~15–25 new):** notable games + redistributable fixture/spec + NES 2.0 metadata
  (per the existing long-tail policy). Each gets register-decode unit tests + boot-smoke/oracle where
  a free fixture exists. Candidates (final list cross-checked against Holy Mapperel + free homebrew):
  NINA-03/06, mappers 34/36/38/41/57/58/60/76/79/113/146/148/232/240/241.
- **Tier 2 (BestEffort, sweep toward 100+):** ported from GeraNES (`ref-proj/GeraNES/src/GeraNES/
  Mappers/Mapper*.h`) / Mesen2; mostly pirate/multicart/homebrew with **no redistributable fixtures**.
  Register-decode unit-tested only; **excluded from the AccuracyCoin/oracle gate**.

  **Honesty is CI-enforced** (the crux of the constraint): a new test asserts every ROM in the
  committed oracle/AccuracyCoin corpus resolves to a non-`BestEffort` mapper — a Tier-2 board can
  never silently enter the accuracy gate. Tier-2 mappers are still pure, deterministic, save-state-
  round-trip-tested; we simply don't *claim* hardware-accuracy. Port in batches into a new
  `crates/rustynes-mappers/src/sprint5.rs` (matching the `sprint{1..4}.rs` grouping), reusing the
  existing bus-conflict helper, `MapperCaps` hook gating, and the MMC3-style A12 IRQ patterns.

**A2. Mapper-89 (Sunsoft-2) render fix** — *Tenka no Goikenban: Mito Koumon*. Concrete root-cause
lead: GeraNES `Mapper089.h` models a **bus conflict** (`data &= readPrg(addr)`) that RustyNES's
`crates/rustynes-mappers/src/sunsoft2.rs` omits; also GeraNES masks banks with a power-of-two mask vs
RustyNES's `% bank_count`. Pin first via a new `crates/rustynes-test-harness/src/bin/capture_m89.rs`
(clone of `capture_left_edge.rs`; dumps frame-450 framebuffer hash + per-frame CHR-bank/mirroring/
PPUMASK trace), then fix in priority order: (a) model the bus conflict like `gxrom.rs`; (b) bank-mask
vs modulo; (c) PPUMASK rendering-enable timing. Add a bus-conflict unit test mirroring
`cnrom.rs::cnrom_bus_conflict_masks_write_with_prg_byte`; flip `docs/compatibility.md` §m89 to resolved.

**A3. GxROM/SMB3 "Mario flashing"** — repro-first triage via
`crates/rustynes-test-harness/src/bin/repro_gxrom_smb3.rs`. Note the report likely misfiles the
mapper (SMB3 is MMC3/m4, not GxROM-66) — verify the actual dump. If not reproducible across a long
run, document as not-a-bug with the harness invocation; ship no code change without a pinned failure.

**A4.** New **ADR 0011 — mapper tiering** (the two-tier policy + the CI invariant). Extend
`docs/mappers.md` + `docs/STATUS.md` matrices with a Tier column and an honest top-line summary;
badge the loaded game's tier in the mapper debug panel.

### B — Library & ROM ingest  → beta.1

**B1. ROM ingest: ZIP + soft-patching.** Single native chokepoint:
`crates/rustynes-frontend/src/app.rs::load_rom_from_path`, immediately after `fs::read`, **before**
format detection — so one deterministic byte buffer flows into `Nes::from_rom*` and save-states /
netplay / oracle see exactly the patched image. Stages: (1) if `.zip`, extract the single
`.nes/.fds/.nsf` entry (picker on multiple) via the `zip` crate; (2) auto-apply a same-stem
`.ips/.ups/.bps` (precedence bps>ups>ips; UPS/BPS verify their in-format CRCs) via a new
`crates/rustynes-frontend/src/patch.rs` (pure `apply_*(rom, patch) -> Result<Vec<u8>>`). The wasm
load path (`AppEvent::RomLoaded`) gets the same preprocessing.
**Hash-keying decision: key the per-game DB off the POST-patch CRC32** — the patched image is the
game that actually runs; consistent with save-state/netplay/oracle identity and with
`game_db.rs::rom_crc32` (PRG+CHR, header-excluded). Mismatched-patch netplay peers then fail fast
instead of desyncing.

**B2. Per-game config database — schema extension.** Extend the existing CRC32 mirroring DB
(`crates/rustynes-frontend/src/game_db.rs`, vendored TetaNES `game_database.txt`) from `(crc,
Mirroring)` to a `GameDbEntry { mirroring, region, mapper/submapper, palette/Vs-PPU, input,
bus_conflicts, title }`. Keep the vendored file as a read-only base; add a **user-overlay file** in
the config dir that the editor writes and that overrides the base by CRC (vendored file never edited
in place). Switch the cache from `OnceLock` to `RwLock`/`arc-swap` so the editor can hot-reload.

**B3. RomOverrides plumbing (firewall preserved).** Mapper/region overrides need a parse-path hook.
Add `Nes::from_rom_with_overrides(bytes, RomOverrides)` where the **frontend** populates `RomOverrides`
from the DB; the core test harnesses keep calling plain `from_rom` (≡ `RomOverrides::default()`), so
oracle runs are structurally identical to today. Extend `app.rs::apply_game_db` to apply the new
fields through existing core setters (`set_mirroring_override`, region setter, `set_vs_ppu_type`,
`set_four_score`, …). Add a CI/doc assertion that the harness imports neither `game_db` nor the overlay.

**B4. In-app ROM-DB editor** (GeraNES's standout feature, Tools → Advanced → ROM Database). New
`crates/rustynes-frontend/src/debugger/game_db_panel.rs` + `ToolPanel::GameDb` in `debugger/mod.rs`,
following `cheat_panel.rs`. Edit the current ROM's entry (dropdowns for mirroring/region/palette/
input/mapper), Save/Reset/Delete to the user overlay (atomic write), hot-reapply via `apply_game_db`.
**Netplay uses the vendored base + header only** (ignores the local user overlay) so per-peer edits
can't desync — documented as a deliberate decision.

### C — Video: shaders + HD-packs  → beta.2

**C1. NTSC per-knob tuning (quick win, the documented follow-up).** Promote the Bisqwit
`CONTRAST`/`SATURATION`/`BRIGHTNESS`/hue from baked WGSL consts to the existing per-frame uniform
(the YIQ→RGB matrix is linear in contrast and contrast·saturation; phase is already a live uniform).
Add `[graphics] ntsc_*` fields (`#[serde(default)]` returning today's hardcoded values → byte-identical
for existing configs) + sliders in `settings_panel.rs::video_section`, pushed live like the existing
`crt_scanline` effect. Output-only; the `index_framebuffer` source is untouched.
Files: `crates/rustynes-frontend/src/ntsc_bisqwit.rs`, `config.rs`, `debugger/settings_panel.rs`.

**C2. Composable shader stack + preset bank** (GeraNES `ShaderPass` model). Refactor the
single-select `if/else` chain in `crates/rustynes-frontend/src/gfx.rs` into a `ShaderStack`
(`Vec<ShaderPass{id, enabled, params}>`) rendered via **ping-pong between two NES-res intermediate
RTs**; an empty stack falls through to today's direct blit (byte-identical default). Shaders declare
knobs via a `#pragma parameter` header parsed to drive generic sliders (mirrors GeraNES
`ShaderWindowUI.inl`). The Bisqwit pass is special-cased (consumes the `R16Uint` index texture; must
be first if present). Ship a built-in CRT preset bank + Save/Load/Delete preset UX. New module
`crates/rustynes-frontend/src/shader_pass.rs` beside `crt.rs`/`ntsc*.rs`.
*Rejected:* a full RetroArch `.slangp` importer (GLSL→WGSL translation is out of scope).

**C3. HD-pack / mod loader — minimal first cut (riskiest item; flag stretch parity).** Mesen-style
`hires.txt`. Needs tile identity the PPU doesn't export today, so add a **feature-gated (`hd-pack`,
off by default) per-frame tile-record export** in `crates/rustynes-ppu/src/ppu.rs` alongside
`index_framebuffer` (one record per BG tile slot + per sprite: `{chr_addr/bank, palette index, flip}`
captured at fetch time) — pure output-only telemetry like `index_framebuffer`/`state_trace`, **must be
proven byte-identical against AccuracyCoin with the feature on and off**. Frontend
`crates/rustynes-frontend/src/hdpack.rs`: load a folder/ZIP `hires.txt`, parse only `<scale>`/
`<patternTable>`/unconditional CHR-hash `<tile>` rules, per-game enable keyed on `rom_sha256`, a
Mod → Load menu entry. Substitution happens at blit time in the frontend (hash each tile's CHR bytes
CRC32 Mesen-compatibly, composite the hi-res replacement; misses fall back) — a `ShaderStack` pass
consuming the tile map. **Stretch, NOT v1.2.0:** memory-condition gating, background-region
replacement, palette-key matching, HD audio.

### D — Input & peripherals  → beta.2

**D1. Family BASIC keyboard** (long-standing follow-up). New `InputDevice::FamilyKeyboard` in
`crates/rustynes-core/src/input_device.rs` implementing the 72-key matrix scan (`$4016` row/col
select, `$4017` read) + a `Nes` setter mirroring `set_power_pad`. Frontend: extend `ExpansionDevice`
(`config.rs`), add key-state to `FrameInputs`/`SharedInput` (`#[serde(default)]`), map host keys in
`input.rs`. `ExpansionDevice::None` stays default → byte-identical.

**D2. Additional peripherals (cheap, high-value).** **SNES mouse** (serial protocol) and **Arkanoid
on both ports** (reuse the existing `VausState` — allow it on `$4016` too) as new `InputDevice`
variants. **Defer** the niche/overlapping ones (Konami/Bandai Hyper Shot, Family Trainer ≈ Power Pad,
Subor keyboard ≈ Family BASIC) to a follow-up — each is an independent variant, additive later.

**D3. Game Genie code-name database.** Bundle a ROM-indexed DB (public Galoob/community lists)
mapping `rom_sha256`/name → `{name, code}`; in the cheat panel, present a pick-list for the loaded
ROM that feeds the existing `genie.rs` decode + `cheats.rs` persistence. Pure frontend; core
substitution unchanged. (Power Pad rebindable keys already exist on native.)

### E — Lua scripting depth  → beta.2

**E1. `emu.onNmi` / `emu.onIrq`** — *unblocked*. Tap the existing
`crates/rustynes-core/src/bus.rs::notify_irq_service(vector, is_nmi)` commit-point (already feeding
the IRQ trace), not the speculative `poll_nmi`/`poll_irq` sampler that ADR 0010 flagged. Add a
`debug-hooks`-gated per-frame interrupt-service log on `Nes` (cleared at `run_frame` top, like
`exec_log`), expose `interrupt_log()`/`set_interrupt_logging()`, and register/replay `onNmi`/`onIrq`
in `crates/rustynes-script/src/lib.rs` exactly like `onExec`. Output-only; off by default.

**E2. `emu.setInput`** — apply through the deterministic late-latch (currently dropped at
`app.rs:2682`). Store a per-port override in the engine, drain it in `pump_scripts`, merge into
`SharedInput`/`FrameInputs` at `EmuCore::latch` (before `produce_one_frame`) so replay stays
bit-identical. **Gate identically to `emu.write`**: reuse the existing `set_writes_locked`
(`netplay_locked || movie_locked`) + `ra_hardcore_blocks()` so it's a no-op in netplay/TAS/RA-hardcore.

### H — Menu-bar UX / responsiveness (GeraNES-inspired)  → beta.2

RustyNES already has a full egui menu-bar shell (`ui_shell.rs`: File/Emulation/Tools/View/Debug/Help
+ Open Recent + save-slot radios + the safe return-`MenuAction`-dispatch-after-the-egui-pass design),
so this is **polish, not a rebuild**. GeraNES's `GeraNESApp.MenuUI.inl` is more responsive in three
concrete ways we don't yet have:

**H1. Per-item contextual enable/disable.** GeraNES greys menu items from live state
(`hasRomLoaded`, `netplayRomChangeRestricted`, `replayInteractionLocked`, `replayRecordingActive`,
`netplayClientRestricted`). RustyNES only reflects a couple of TAS flags today; thread the same
predicates into `MenuShellState` and pass `enabled` to each `ui.add_enabled(...)`/`ui.menu_button`
so e.g. Save/Load-state, Close-ROM, region/mapper items disable with no ROM, during netplay, or
mid-replay. Pure UI; no determinism surface.

**H2. Inline keyboard-shortcut accelerator labels + a remappable shortcut registry.** GeraNES shows
the bound key next to each item from a configurable `m_shortcuts` map. Add a `[shortcuts]` config
section (serde-default = today's hardcoded hotkeys → byte-identical configs) feeding both the global
hotkey handler and inline accelerator labels in the menu (egui supports a shortcut-text column).
Keyboard-hotkey remapping mirrors the existing gamepad-rebinding UI.

**H3. (optional) FontAwesome-style menu icons.** GeraNES decorates every item with an icon glyph.
Lower priority + a font-asset cost; gate behind a setting or defer if it bloats the wasm size budget.

*Rejected:* GeraNES's `CustomWindowChromeUI` (self-drawn titlebar/chrome) — RustyNES keeps native OS
window decorations on desktop. Files: `crates/rustynes-frontend/src/{ui_shell.rs,config.rs,input.rs,
app.rs,debugger/settings_panel.rs}`.

### F — Web / wasm parity  → beta.3

**F1. Touch controls (best web win).** Pointer-Events overlay (translucent DOM/CSS in
`crates/rustynes-frontend/web/index.html` → minimal wasm-size cost) producing a `Buttons` mask routed
to a selectable target port (Port1/2/expansion/multitap), consumed by both wasm frontends
(`wasm.rs` canvas embed → `set_buttons`; `wasm_winit.rs` → `InputState`). Latched at the identical
late-latch point → no new determinism surface, recorded by TAS/netplay like a keypress. Add
`PointerEvent`/`TouchEvent` to the wasm `web-sys` features.

**F2. Power Pad wasm feed.** Native works; wasm doesn't. In `wasm.rs`, on Power-Pad selection call
`set_expansion_device(1, PowerPad)` + feed `set_power_pad(1, mask)` each rAF tick (mask from the touch
overlay or `POWER_PAD_KEYS`). In `emu.rs`, narrow the `#[cfg(not(wasm32))]` so the Power-Pad latch arm
(needs only a `u16` mask) compiles on wasm; keep Zapper/Vaus native-gated.

**F3. Hosted netplay (deploy + verify, not new code).** Browser netplay is **already code-complete**
(`wasm_netplay.rs` + `wasm_lobby.rs` + `rustynes-netplay/{webrtc,signaling}.rs` + the `deploy/` Docker
bundle; COOP/COEP is *not* a blocker — DataChannel + the AudioWorklet need no SharedArrayBuffer).
Stand up `deploy/` on a real host/domain (Caddy TLS + coturn), bake the public `signaling_url`/STUN/
TURN into the shipped `[netplay]` defaults, run the manual 2-tab → 2-machine → 4-player matrix, and
flip `docs/netplay-webrtc.md` §4 Pending→Verified. Polish: surface connection-quality/desync status in
the lobby. *Caveat:* TURN relay bandwidth is an ongoing ops cost (flag, not code).

**F4. Experimental wasm Lua (piccolo backend, off by default).** Refactor `rustynes-script` to a
VM-binding trait behind the already-clean `emu.*` host surface + `ControlCmd`/`DrawCmd`/log queues;
native keeps mlua, wasm gets a pure-Rust **`piccolo`** backend behind a new off-by-default
`script-wasm` feature (piccolo's fuel ≙ the existing per-frame instruction budget). **Documented as
experimental and explicitly NOT byte-identical to native Lua** — fine because scripts are
observational/overlay + gated writes, never part of the FB/audio determinism oracle. New **ADR 0012**
records the divergence. Must pass `scripts/wasm_size_budget.sh`.

**Deferred (per maintainer decision): browser RetroAchievements → v1.3 backlog.** rcheevos is C
(needs an emscripten fork to link on wasm), the RA `User-Agent` header is browser-forbidden, and
hardcore integrity is weaker in a browser. Tracked, not built, in v1.2.0.

### G — Performance / PGO  → any beta (independent infra)

Gate the existing complete-but-unused `scripts/pgo/run.sh` into a **release-/manual-dispatch-only** CI
job (not per-PR — instrument+train+rebuild is too slow): build the PGO `rustynes` binary against the
committed CC0/MIT training ROMs, then **promote only if it beats plain release by the >3%
Criterion-stable threshold** (`cargo bench -p rustynes-core --bench full_frame`). Add **BOLT** (Linux)
as an optional second stage behind the same gate. Apply only doc-flagged micro-opts that *measure* as
>3% wins and respect the hot-path rules. No core behavior change.

## Verification (every beta + finals)

- **Hard gate:** `cargo test --workspace --features test-roms` → AccuracyCoin 100% (139/139),
  60-ROM + extended oracles byte-identical, nestest 0-diff. Re-run with `hd-pack`/`debug-hooks`
  **on** to prove byte-identical telemetry.
- **CI honesty gate (new):** every oracle/AccuracyCoin ROM resolves to a non-`BestEffort` mapper;
  the test harness imports neither `game_db` nor the user overlay.
- **Quality gates:** `cargo fmt --check`; `clippy --workspace --all-targets -D warnings` (+ `scripting`
  + both wasm flavours + new `script-wasm`); `RUSTDOCFLAGS=-D warnings cargo doc`; the `no_std`
  `rustynes-core` cross-compile; `pre-commit run markdownlint --all-files` (v0.39.0 gate).
- **Per-feature:** register-decode unit tests for every new mapper + save-state round-trip;
  m89 framebuffer-hash regression; screenshot-corpus regression after shader/HD-pack work; Lua API +
  sandbox + setInput-gating tests (no-op under netplay/TAS/RA-hardcore); patched-ROM determinism
  (save-state + netplay identity on post-patch CRC).
- **Web:** wasm clippy + `trunk build` (both `wasm-winit`/`wasm-canvas`) + `wasm_size_budget.sh`;
  keep `Trunk.toml` wasm-bindgen pin == `Cargo.lock`; the manual live-netplay matrix (can't be
  CI-verified) recorded as a checklist.
- **Perf:** PGO/BOLT promoted only on the >3% Criterion-stable bar.
- **Manual:** `cargo run --release -p rustynes-frontend` — exercise each filter/preset/HD-pack,
  each new peripheral + Family BASIC keyboard, the ROM-DB editor, ZIP/patched loading, a Genie
  name-DB code, the Lua onNmi/onIrq + setInput, and confirm netplay/TAS still gate writes.

## Out of scope / deferred backlog (documented, not built)

Browser RetroAchievements (v1.3); full Mesen HD-pack parity (conditions/background-replacement/HD
audio); the niche peripherals (Hyper Shots / Family Trainer / Subor); Vs. DualSystem two-CPU/PPU games;
Kid Icarus FDS side-B post-registration interactivity; mobile (iOS/Android) frontends; the hard-tier
C1 IRQ-sample-point residuals (`cpu_interrupts_v2`, `mmc3_test_2/4` #3, two `apu_reset` cases — explicit
*document-don't-grind* per policy); the aggressive mapper sweep beyond ~100 toward the full ~300.
