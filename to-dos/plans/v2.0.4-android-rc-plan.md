# v2.0.4 "Harbor" — Android Release Candidate (sprint plan)

> **Scope: the Android release-candidate closeout of the v2.0.x mobile-finalization
> train.** A **host / Android-only** cut — the cycle-accurate emulation core is
> **unchanged and byte-identical to v2.0.3** (AccuracyCoin **141/141, 100.00%**;
> nestest 0-diff; the `#![no_std]` chip stack untouched), so no accuracy / save-state /
> determinism number moves. v2.0.4 stages the RC scaffolding a maintainer needs to
> upload the Android app onto a Play Console testing track, and clearly flags every
> step that can only run on a device. The `foss` flavor stays **behaviour-identical**.

## Plan reconciliation

The `v2.0.x-mobile-finalization-plan.md` roadmap slots **v2.0.4** as the "Android
release candidate" and **v2.0.3** as "polish + performance". In practice the accuracy
work (ADR 0030 "Option 1", the 2-cycle-ALE fetch refactor) grew into a Timebase-scale
campaign and **landed in v2.0.3** — the shipped default reached AccuracyCoin 141/141
there. v2.0.4 is therefore the pure **Android RC** cut: no core work, only the host-
authorable release scaffolding + config + docs, with the on-device gates flagged for
the maintainer / v2.0.9. This is the intended end state of the "Android final
v2.0.1 → v2.0.4" window.

## Host-done (this release)

- **Version bump.** Workspace `2.0.3 → 2.0.4` (cascades version-only through
  `Cargo.lock`); Android `versionCode 20003 → 20004`, `versionName "2.0.3" → "2.0.4"`.
- **Release signing scaffold.** `release` build type always resolves a signing config:
  the upload keystore (`keystore.properties` file or `RUSTYNES_UPLOAD_*` env vars —
  maintainer secret, gitignored / CI-injected) when present, else a **graceful
  fallback to the debug signing config** so a keyless CI / local
  `assemble{Foss,Play}Release` still yields an installable (debug-signed, never
  shippable) RC artifact. `keystore.properties` / `*.jks` / `*.keystore` were already
  gitignored (v2.0.3) — verified, no change needed.
- **Crash / ANR hardening (code).** `DebugStrictMode` — a debug-only, log-only,
  `BuildConfig.DEBUG`-guarded StrictMode installer (thread: main-thread disk / network
  I/O + slow calls; VM: leaked closeables / SQLite / receivers), installed first in
  `MainActivity.onCreate`, shared `src/main` (foss + play), inert in release.
  Complements the existing opt-in `CrashReporter` (unchanged).
- **R8 / ProGuard final hardening review.** Release keep set re-audited and confirmed
  complete; no keep loosened; audit outcome recorded in `proguard-rules.pro`.
- **Fastlane / Play Console listing metadata.** `fastlane/metadata/android/{en-US,es-ES}/`
  — `title.txt` / `short_description.txt` / `full_description.txt` /
  `changelogs/20004.txt`, within Play field limits. Files only; no upload.
- **Release docs.** `CHANGELOG.md` `[2.0.4]`; `docs/STATUS.md` current-release block
  (v2.0.3 demoted); `to-dos/ROADMAP.md` status; `.github/release-notes/v2.0.4.md`;
  this plan; the mobile-finalization plan's v2.0.4 bullet.

## Flagged for maintainer / on-device (v2.0.9 / the RC upload)

Inherently on-device or maintainer-secret steps — **not** performed here (no device in
the build environment; `adb` empty):

- Upload the AAB to the Play Console **internal / closed testing track**.
- Sign with the **real upload keystore** (inject the secret); confirm Play App Signing
  enrollment.
- Publish the staged store listing via fastlane / the Play Console.
- **Crash-free-rate + ANR gate on real hardware** (Android vitals: crash < 1.09%,
  ANR < 0.47%, per-device < 8%).
- **Live monetization runtime** verification (AppLovin MAX + RevenueCat with real keys
  / Play Console products) — dormant until then.
- **Deferred per-feature gate migration (do NOT do on host):** keep the existing
  Play-Billing `LicenseManager` freemium gates (save-states / fast-forward / …) as-is;
  rewiring them to the monetization `feature_enabled` façade is a product-architecture
  decision needing on-device validation — deferred to **v2.0.9**.
- Regenerate the Baseline / Startup Profile on-device against the v2.0.0 core (v2.0.9).

## Verification gates (host)

- `cargo check --workspace` (version bump + lock regen compiles; version-only lock
  diff), plus `cargo fmt --all --check`.
- Android `:app:compileFossDebugKotlin` + `:app:compilePlayDebugKotlin` both BUILD
  SUCCESSFUL (the real gate for the signing-config + `DebugStrictMode` changes);
  `:app:assembleFossRelease` if time permits (validates the debug-signing fallback path).
- `pre-commit run markdownlint --all-files` on the new / edited docs.
