## Context

RustyNES shipped a complete **Android** app as an iterative train, **v1.8.0
"Android"** (the UniFFI `rustynes-mobile` foundation) → … → **v1.8.8 "Atlas"**
(Play-readiness) → **v1.8.9 "Backlog"** (creator tooling + the dormant
monetization build-out), each release additive / off-by-default so the
cycle-accurate core stayed byte-identical and AccuracyCoin held 139/139. The
maintainer wants the **iOS/iPadOS app built the same way** — a 10-release
**v1.9.0 → v1.9.9** train that mirrors the Android arc release-for-release, on the
same v1.0.0 production core, distributed by **TestFlight** until the v2.1.0 joint
store launch.

**TL;DR feasibility — de-risked by precedent.** Apple's **April-2024 §4.7**
permits retro-console emulators worldwide, and **Delta** (NES-capable) shipped
and *topped* the App Store with user-provided ROMs. RustyNES is a **pure
interpreter** → it needs **no JIT**, the one capability iOS withholds — so the
single hardest iOS-emulator constraint simply does not apply. NES ROMs have **no
encryption**, so the legal posture is clean post-*Nintendo v. Yuzu*.

## Hard contract (unchanged; applies to every v1.9.x below)

- **The chip stack + the emitted-frame/sample contract are NEVER touched.** All
  iOS work lives in a `rustynes-ios` shim + a SwiftUI shell over the shared
  `rustynes-mobile` bridge. The desktop / native / `no_std` / wasm builds stay
  **byte-identical**, and **AccuracyCoin 100% (139/139)** + the oracles stay green
  on the existing host CI.
- **Determinism holds on ARM** (same seed+ROM+input ⇒ bit-identical FB+audio,
  proven by AccuracyCoin 139/139 on Android ARM64 CI). A state saved on
  desktop/Android loads on iOS, and a `.rnm` TAS replays identically, so
  desktop ⇄ Android ⇄ iOS netplay cross-play stays valid. The DRC resampler +
  run-ahead stay **frontend-side** (timing-only), never in core synthesis.
- **NEVER bundle commercial ROMs.** Zero copyrighted ROMs in the `.ipa`. ROM
  import is **user-provided only** — Files / `UIDocumentPicker` / share-sheet /
  iCloud Drive — copied into the app sandbox keyed by SHA-256. Optionally bundle
  only CC0 / public-domain homebrew demos.
- **No JIT anywhere.** The CPU/PPU/APU core is a cycle-stepped interpreter; `mlua`
  is interpreted Lua 5.4 (not LuaJIT); wgpu→Metal shaders are compiled by Apple's
  toolchain at build/load. No `--all-features` (mlua vs piccolo are mutually
  exclusive). `unsafe` is confined to `rustynes-ios` + the FFI glue, each with a
  `// SAFETY:` note.

## The reused architecture (iOS inherits almost everything)

The mobile stack is already cross-platform by design (ADR 0024). iOS adds only a
thin shim + a SwiftUI shell — the iOS analog of `rustynes-android` + `android/`:

- **`rustynes-core`** — the deterministic `#![no_std]`+alloc chip stack.
  **Untouched.** Byte-identical on ARM; a pure cycle-stepped interpreter, so iOS's
  **no-JIT / W^X** rule is a **non-issue**.
- **`rustynes-mobile`** (UniFFI bridge) — **no changes needed.** UniFFI generates
  **Swift** bindings exactly as it generates Kotlin
  (`uniffi-bindgen-swift` / `cargo swift package`), packaged as an
  **`.xcframework`**. The typed surface (`nes_new`, `run_frame`, `drain_audio`,
  `set_controller`, `snapshot`/`restore`, plus the v1.8.6 `load_script` / `ra_*` /
  `np_*`) is Swift-ready verbatim.
- **Shared crates reused verbatim:** `rustynes-script` (mlua — interpreted Lua
  5.4, **not** LuaJIT → iOS-legal), `rustynes-ra`, `rustynes-netplay`,
  `rustynes-gfx-shaders` (WGSL → Metal via wgpu), `rustynes-hdpack`,
  `rustynes-monetization` (dormant until v2.1.0). The `resampler.rs` lock-free
  ring + Hermite DRC and the `.rns` save-state format are platform-agnostic
  (cross-device + cross-play valid).
- **NEW `crates/rustynes-ios`** (the iOS shim, mirroring `rustynes-android`) — a
  `staticlib` carrying only the hot glue UniFFI can't express: the
  **`CAMetalLayer` → wgpu-Metal** surface handoff and the **CoreAudio /
  AVAudioSession** sink. `unsafe` confined + `// SAFETY:`-documented. ADR 0026.
- **NEW `ios/` SwiftUI app** (the iOS analog of `android/`) — an Xcode/SwiftPM
  project consuming the xcframework: the scene/lifecycle, the Metal
  `MTKView` / `CAMetalLayer` host, the on-screen controller + GameController
  input, `UIDocumentPicker` ROM import, settings, and the per-version feature
  shells.

**Determinism boundary holds:** the core's `run_frame()` output is untouched;
late-latch input flows through the existing `SharedInput` model; DRC + run-ahead
are frontend stages. iOS supplies only surface / audio-sink / lifecycle / storage
/ UI.

## v1.9.0 "Sunrise" — Foundation & MVP  (≈ Android v1.8.0)

The iOS analog of v1.8.0: stand up the shim + app + build, then port the focused
MVP (core + video + audio + input + persistence). Lua / RA / netplay exist in the
bridge but are **not surfaced yet** (v1.9.6).

### Workstreams

- **A — Foundation: build + shared bridge + signing  (L).** Add
  `crates/rustynes-ios` (`staticlib`) over the shared `rustynes-mobile` UniFFI
  surface. `rustup target add aarch64-apple-ios aarch64-apple-ios-sim`; UniFFI +
  `cargo swift package`; `xcodebuild -create-xcframework` (lipo the sim arches);
  a checked-in SwiftUI Xcode/SwiftPM project consuming the xcframework. Apple
  Developer Program, App ID, distribution cert + provisioning profiles,
  **fastlane match** (certs in a private git repo). GitHub Actions `macos-latest`
  + fastlane (gym/match/pilot), **gated to tags + manual dispatch** (macOS minutes
  bill ~10×, mirror the PGO/release gating). *Determinism-safe* (host shim only).
- **B — Metal rendering + surface  (M).** Start with a ~30-line hand-Metal
  nearest-filter blit of the 256×240 RGBA texture to `CAMetalLayer` — fast,
  predictable on old devices, lighter than wgpu for a single textured quad.
  Letterbox to the NES **8:7 PAR** reusing the `gfx.rs` math. (The full
  **wgpu→Metal** path graduates in v1.9.4.) *Determinism-safe* (output-only).
- **C — CoreAudio  (M).** cpal CoreAudio host + **AVAudioSession**
  (`objc2-avf-audio`); reuse the lock-free SPSC ring + Hermite DRC **verbatim**
  (`resampler.rs`). AVAudioSession `.playback` category, request a small
  `preferredIOBufferDuration`. *Determinism-safe* — DRC is timing-only.
- **D — Touch + controller input  (M).** SwiftUI touch overlay reusing the proven
  wasm overlay layout (D-pad / A / B / Start / Select + target-port selector);
  **GameController** (`GCController`) discovery (MFi / Xbox / DualSense
  auto-discovered). All inputs converge on the same `set_controller` late-latch
  bit-mask → TAS/netplay-identical.
- **E — Storage + lifecycle  (M).** ROM import via `UIDocumentPicker` /
  `.fileImporter` / share-sheet + **security-scoped bookmarks**; copy into
  `Application Support/RustyNES/{roms,saves}/<sha256>/`. `.rns` save-states +
  battery SRAM (format platform-independent → cross-device portability). SwiftUI
  `ScenePhase` pause/resume. `PrivacyInfo.xcprivacy` (no data collected).
- **F — MVP feature port  (L).** Ports cleanly: save-states / rewind / run-ahead,
  cheats (Game Genie / raw RAM), TAS `.rnm` **playback**. **Deferred** (surfaced
  later): Lua console, RA, netplay (all already in the bridge).

### Exit criteria

A signed TestFlight build boots a homebrew ROM on a physical iPhone (14+) and iPad
at 60 Hz; audio plays with no underruns through a backgrounding round-trip; touch
+ an MFi controller both drive P1; ROM import + save-states survive an app restart;
the privacy label reads "Data Not Collected"; the **first §4.7 review passes**
(~1–2 h). Host CI stays green; AccuracyCoin 139/139.

### Sprint sketch

1. M0 spike (winit verbatim → a pixel in the Simulator) → A foundation (shim +
   xcframework + signing + CI).
2. B Metal blit + C CoreAudio (alpha.1 boots in Simulator).
3. D input + E storage/lifecycle on-device (alpha.2 / beta.1).
4. F MVP port + the §4.7 metadata pass → TestFlight (rc.1 / v1.9.0).

### Reused vs net-new

- **Reused:** `rustynes-core`, `rustynes-mobile` (verbatim), `resampler.rs`, the
  `gfx.rs` 8:7/letterbox math, the wasm touch-overlay *design*, the `SharedInput`
  late-latch model, the `.rns` format.
- **Net-new:** `crates/rustynes-ios` (the shim — ADR 0026), the `ios/` SwiftUI
  project, `fastlane/` (match/gym/pilot), `.github/workflows/ios.yml` (tag-gated),
  `docs/ios.md`.

### Re-research before starting

§4.7 + §4.7.1–§4.7.5 current text; the App-Store-Connect TestFlight beta-review
requirements; the current Xcode / iOS-SDK submission floor; UniFFI + `cargo swift`
+ `xcodebuild -create-xcframework` current invocation; `objc2-avf-audio` AVAudioSession
API.

## v1.9.2 — Input & the on-screen controller  (≈ Android v1.8.2)

The iOS analog of the v1.8.2 priority increment: fix multi-touch + draw a styled
NES-001 controller, both resizing/remapping correctly across iPhone/iPad.

### Workstreams

- **A — Multi-touch (the SwiftUI analog of the Compose fix).** Replace per-button
  tap gestures with **one container-level multi-pointer responder** (a SwiftUI
  `DragGesture(minimumDistance: 0)` on a `simultaneousGesture`, or a `UIView`
  multi-touch responder bridged via `UIViewRepresentable`) that tracks **every
  touch by identity** and recomputes the entire `Buttons` mask from all
  currently-pressed touches each event — giving simultaneous presses, D-pad
  diagonals, and slide-roll for free, all flowing through the existing late-latch.
  (The Android cut's bug — dropping held fingers when a 3rd touches down — is the
  warning: track the full pointer set, not per-event deltas.)
- **B — The virtual NES controller art.** Render the NES-001 pad from the
  `make_icon.py` `nes_controller()` geometry/palette (SwiftUI `Canvas` or a
  converted vector asset), with pressed-state lighting driven by the live mask.
- **C — Responsive sizing + hit-region remap.** Derive every button region from
  the **measured controller bounds** (a `GeometryReader` rect) as the same
  fractions the art uses, so a resize moves art + regions in lockstep. Per-idiom
  layouts (compact iPhone / large iPad / split-view / Stage Manager), a
  size/opacity setting.
- **D — GameController depth + haptics.** P1–P4 mapping (MFi / Xbox / DualSense),
  in-app remap, **Core Haptics** light press feedback (off by default).

### Exit criteria

On-device: A+B+direction register simultaneously; diagonals work; a press in a
resized layout hits the right button; a hardware controller and the overlay both
drive P1; haptics toggle works. AccuracyCoin 139/139.

### Sprint sketch

1. A multi-touch responder + late-latch wiring (the bug-prone core).
2. B art + C responsive remap.
3. D GameController P1–P4 + remap + haptics → TestFlight.

### Reused vs net-new

- **Reused:** the wasm/Android touch region model + the `make_icon.py`
  `nes_controller()` geometry; the `set_controller` late-latch.
- **Net-new:** the SwiftUI multi-touch responder, the `Canvas` controller art, the
  GameController remap UI, the Core Haptics layer.

### Re-research before starting

Current `GameController` framework capabilities (DualSense / Backbone support);
SwiftUI multi-touch best practice (responder vs `spatialEventGesture`); Core
Haptics API.

## v1.9.4 — Metal renderer + shader stack  (≈ Android v1.8.4)

The iOS analog of v1.8.4: graduate from the hand-Metal blit to the full
**wgpu→Metal** render path + the shared WGSL shaders.

### Workstreams

- **wgpu→Metal render path.** wgpu Metal surface from the `MTKView`'s
  `CAMetalLayer`; reuse the desktop WGSL **NTSC / CRT / Bisqwit** stack via the
  shared **`rustynes-gfx-shaders`** crate (WGSL → Metal at build/load); reuse the
  8:7-PAR / overscan / letterbox blit. Retires the v1.9.0 hand-blit baseline (kept
  as a fallback for the oldest devices).
- **CADisplayLink pacing.** ProMotion 120 Hz (`CADisableMinimumFrameDurationOnPhone`);
  emulate at the console rate **60.0988 Hz**, present 1:1, DRC absorbs the beat
  (reuse the existing display-sync logic).
- **Surface-loss + lifecycle.** Recreate the surface on resize / scale /
  drawable-loss; drop the drawable on background, rebuild on foreground; handle
  audio-route + interruption notifications alongside.
- **Native-audio hot path.** Drain audio as bytes across the FFI (no per-frame
  boxing) — the iOS analog of the v1.8.4 native-audio JNI hot path.

### Exit criteria

The WGSL shader stack renders on-device at parity with desktop; ProMotion devices
pace smoothly with no audible DRC artifacts; backgrounding + rotation + external
display attach/detach never black-screen or crash. AccuracyCoin 139/139 (the
core/output is untouched — presentation only).

### Sprint sketch

1. wgpu→Metal surface from `CAMetalLayer` + the 8:7 blit.
2. The WGSL shader stack via `rustynes-gfx-shaders` + CADisplayLink ProMotion.
3. Surface-loss/lifecycle hardening + the native-audio hot path → TestFlight.

### Reused vs net-new

- **Reused:** `rustynes-gfx-shaders` (WGSL), the desktop wgpu present path + the
  display-sync/DRC logic, the `gfx.rs` blit math.
- **Net-new:** the `CAMetalLayer`→wgpu surface handoff (in `rustynes-ios`, ADR
  0026), the CADisplayLink pacer, the iOS surface-loss lifecycle.

### Re-research before starting

wgpu's current `CAMetalLayer` / Metal-surface ergonomics; ProMotion / variable-
refresh APIs; AVAudioSession route-change + interruption handling.

## Distribution & §4.7 compliance (the research-backed strategy)

- **§4.7 (April 5, 2024) green-lights NES emulators on the App Store.** Compliance
  rules baked in from day one: **no bundled / downloadable ROMs** (user-sourced via
  `UIDocumentPicker` only), **no in-app ROM links**, **no Nintendo branding / art**,
  an in-app **ownership notice**, a **searchable library / index** (§4.7.4), an age
  rating (4+ as a tool; gate if M-rated content per §4.7.5). NES ROMs have **no
  encryption** → fully legal post-*Nintendo v. Yuzu*. Precedent: **Delta** ranked
  #1 on the App Store; Provenance + RetroArch also ship.
- **v1.9.0–v1.9.9 = TestFlight only.** The first build gets the full §4.7 review
  (~1–2 h); later builds usually skip review; each build expires after **90 days**
  → re-upload each v1.9.x (the v1.9.1 patch cadence covers it); up to **10,000
  external testers**. **On-device testing required** (iPhone 14+/iPad; the
  Simulator is insufficient — Apple rejects crashes that only appear on hardware).
- **App Store + AltStore PAL launch deferred to v2.1.0** (joint with Android).
  **AltStore PAL** (EU DMA, **notarization-only**, self-hosted, no §4.7 content
  limits) is the secondary channel for the EU + any feature App Review might
  reject. A GitHub-source FOSS build is always available.
- **Monetization** stays dormant through v1.9.x (TestFlight is free + full-featured).
  It activates at **v2.1.0** via the existing `rustynes-monetization` crate + the
  **`foss`/App-Store split (ADR 0025 / ADR 0027)**: the App-Store/`appstore` build
  carries **RevenueCat + StoreKit 2** ($3.99 unlock) + **AppLovin MAX** ads behind
  **ATT**; the `foss` build (AltStore PAL + F-Droid + GitHub) is ad/tracking-free.
  This reconciles the research's "privacy-first launch" recommendation with the
  established ad-supported model — by channel, not by compromise. The privacy
  nutrition label is **"Data Not Collected"** for foss / TestFlight builds.

## Technical stack (from the research; concrete + current)

- **Bindings / build:** UniFFI → Swift (`cargo swift package --release`) →
  `.xcframework` via `xcodebuild -create-xcframework`; targets
  `aarch64-apple-ios` + `aarch64-apple-ios-sim`. The Rust-lib ↔ generated-Swift
  version coupling is the iOS analog of the wasm-bindgen pin discipline.
- **Render:** a ~30-line hand-Metal nearest-blit to `CAMetalLayer` for the v1.9.0
  alpha (fast, predictable on old devices), graduating to the **wgpu→Metal** path
  in v1.9.4 to reuse the WGSL shader stack. `CADisplayLink` +
  `CADisableMinimumFrameDurationOnPhone` for ProMotion; DRC absorbs the
  60.0988 Hz ↔ 120 Hz beat.
- **Audio:** `cpal` CoreAudio + **AVAudioSession** (`.playback`, interruption +
  route-change notifications, the background-audio capability); reuse
  `resampler.rs` verbatim.
- **Input:** **GameController** (auto-discovery, standard mapping) + a SwiftUI
  touch overlay (reuse the wasm-overlay layout) + **Core Haptics**; all
  late-latched on the emu thread.
- **Storage:** `UIDocumentPicker` + **security-scoped bookmarks**; sandbox
  `Application Support/RustyNES/{roms,saves}/<sha256>/`; `.rns` saves; iCloud
  (KV-store config v1.9.5, CloudKit save-state sync v1.9.7).
- **Monetization (v2.1.0):** **RevenueCat** + **StoreKit 2** ($3.99 unlock) +
  **AppLovin MAX** ads behind **ATT** on the App-Store flavor only (ADR 0027).
- **CI:** GitHub Actions **macOS runner**, **tag-gated + manual-dispatch only**
  (macOS is ~10× Linux cost): cargo iOS targets → `cargo swift` → xcframework →
  `xcodebuild` archive/export → **fastlane match** (signing from a private repo) →
  **fastlane pilot** / `upload-testflight-build`.
- **Reference:** the workspace `~/Code/Commercial_Private-Projects/AirGapSync/`
  (a shipping SwiftUI + Rust app) is a concrete in-house pattern for the
  Swift ⇄ Rust boundary, signing, and the macOS CI shape.
