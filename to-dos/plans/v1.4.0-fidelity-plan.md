# RustyNES — Plan: v1.4.0 "Fidelity" (proposed codename)

> Generated 2026-06-16 from research across `nesdev_wiki/`, `ref-proj/GeraNES/`,
> `docs/`, `ref-docs/`, `to-dos/`, and the v1.2.0/v1.3.0 plans. Codename is a
> suggestion (v1.0 "Cycle-Accurate" → v1.1 "Scriptable" → v1.2 "Curator" →
> v1.3 "Bedrock" → **v1.4 "Fidelity"**); rename freely. Theme and stretch items
> are flagged as **maintainer decisions** at the end.

## Context

v1.3.0 "Bedrock" (in flight) delivered foundation + devtools + breadth: toolchain
modernization (edition 2024, Rust 1.96, egui 0.34 / wgpu 29 / rfd 0.17), the
frame-pacing fix, the menu/Settings reorg + Memory Compare + auto-save, the mapper
sweep **87 → 101** + Vs. DualSystem header *detection*, and (still landing) HD-pack
memory-condition gating (E1), peripherals/UX (F), netplay desync diagnostics (G),
a PGO perf gate (H), and casual-mode browser RetroAchievements (I).

**v1.4.0 is the compatibility-and-finish release.** Where v1.3.0 was breadth,
v1.4.0 is depth + polish: close the highest-impact *real-game* accuracy gap that
remains tractable (DMC-DMA controller corruption), harden the test corpus by
wiring the large set of free/committable test ROMs that already exist for features
RustyNES ships, finish the GeraNES-class devtools + audio-mixing UX, bring the
browser build toward native QoL parity, and land a measured, determinism-safe
performance pass. It deliberately does **not** open the v2.0 architectural axis.

**Hard contract (unchanged, applies to everything below):**
- Determinism: same seed + ROM + input ⇒ **bit-identical** framebuffer + audio.
  **AccuracyCoin 100% (139/139)** + the commercial-ROM oracles byte-identical must
  hold after every change. New behavior that *is* an accuracy change (A1/A2/A3)
  pins its failing test-ROM expectation FIRST, then implements until it passes,
  and re-baselines AccuracyCoin only with the new ROM in the gate.
- New features are **off by default / additive** so the shipped, wasm, and
  `no_std` builds stay byte-identical where the feature is off. New devtools live
  behind the existing `debug-hooks`/`hd-pack` features.
- The chip stack stays `#![no_std]` + alloc; only `rustynes-frontend` +
  `rustynes-cheevos` carry `unsafe`. Hot paths: no allocation, fixed arrays,
  profile before abstracting; ≤ 2 ms/frame headless. The core test harness never
  consults any frontend database.

## STANDING RESEARCH DIRECTIVE (maintainer, carry through execution)

Research is **continual**, not just up front. While building each workstream:
- Pin behavior against `nesdev_wiki/` + the committed test ROMs (test-ROM-is-spec);
  when in doubt, read the wiki page cited in the workstream.
- Consult **GeraNES** (`ref-proj/GeraNES/`) for UI/UX + feature shape, **Mesen2**
  (`ref-proj/Mesen2/`) for accuracy reference, and the other `ref-proj/` emulators
  as needed.
- **Fold in beneficial enhancements + determinism-safe performance optimizations
  opportunistically** — even when not explicitly listed here — whenever they
  improve the codebase vs. the current state, provided they respect the hard
  contract and are measured (perf) / test-pinned (accuracy). Surface anything that
  turns out larger than a quick win as its own ticket rather than scope-creeping a
  PR.

## Workstreams

### A — Real-hardware accuracy (the headline compatibility win) → beta.1

**A1. DMC-DMA ↔ controller-read bus conflict** (HIGH real-game impact; the one
genuinely-impactful accuracy gap left that is *not* the v2.0 axis). When a DMC DMA
sample fetch coincides with a CPU read of `$4016`/`$4017`, the 2A03's multiplexed
bus corrupts the read (lost/duplicated controller bits → open-bus). Real games
affected: *Bee 52, Fire Hawk, Mig 29 Soviet Fighter, Ultimate Stuntman, Skate or
Die 2*; *Arkanoid* paddle reads. nesdev ref: `nesdev_wiki/output/DMA.md`
§Register conflicts. Model the DMC-DMA window in `crates/rustynes-core/src/bus.rs`
(`unified_dma_cycle_impl` already tracks the get/put cadence) and return open-bus
for `$4016`/`$4017` reads landing inside the conflict cycles. Currently documented
as a known-open issue (`docs/nesdev-hardware-emulation-checklist.md`,
`docs/compatibility.md`). **Medium.** Pin `read_joy3`/`sprdma_and_dmc_dma` first
(see B1).

**A2. Triangle ultrasonic silence** — halt the triangle channel when its timer
period < 2 (frequency > ~55.9 kHz). *Mega Man 2* (Crash Man) silences the triangle
this way. nesdev ref: `nesdev_wiki/output/APU_Triangle.md`. **Trivial** (gate output
in `rustynes-apu` triangle). Low real-game impact but high polish-to-effort; verify
audio determinism unchanged for the period ≥ 2 path (byte-identical default).

**A3. APU duty-sequencer reset on `$4003`/`$4007` write** — writing the pulse
length/timer-high register resets the duty sequencer phase but not the divider.
Test ROM `apu_phase_reset` (Rahsennor). **Small.** Pin the test first.

*Explicitly NOT in A (kept deferred):* the 3 hard-tier C1 IRQ-sample cases +
`mmc3_test_2/4` #3 + 2 `apu_reset` cases (the v2.0 fractional-master-clock axis,
ADR 0002 stop-condition); OAM DRAM decay + open-bus decay (by-design, no game
depends — document only).

### B — Test-coverage hardening (wire the free/committable test ROMs) → beta.1

RustyNES *implements* far more hardware than it *test-gates*. These suites are
CC0/free and committable to `tests/roms/` (unlike the gitignored commercial
oracle). Each is a regression gate for an already-shipping feature; the effort is
test-harness boilerplate, not core logic. Catalogue from
`nesdev_wiki/output/Emulator_tests.md`.

- **B1. DMC-DMA suite** — `read_joy3`/`count_errors`, `sprdma_and_dmc_dma`,
  `dma_sync_test_v2`. Gates A1 (pin these before implementing A1).
- **B2. PAL APU** — `pal_apu_tests` (the NTSC versions already pass; region-aware
  dividers exist, so this should pass and locks PAL-APU timing).
- **B3. Peripheral gates** — `allpads`, `vaus`/`vaus_test` (Arkanoid 9-bit),
  `Zap Ruder` (Zapper), `mset` (SNES mouse), `FdsIrqTests` (FDS IRQ),
  `characterize-vs` (Vs. 2C03/2C04/2C05 RGB). All for hardware RustyNES already has
  but doesn't CI-gate.
- **B4. Visual/timing** — `240p Test Suite`, `nmi_sync` reference baselines.
- Add a `docs/compatibility.md` §"Edge cases not modeled (by design)" recording
  OAM/open-bus decay + the DMC-DMA model + the Dendy 35464-cycle VBlank race
  (`nesdev_wiki/output/PPU_power_up_state.md` §Dendy).

### C — Audio finish (GeraNES-class per-channel mixing) → beta.2

**C1. Per-channel audio mixing UI.** Today RustyNES has the 5-band EQ (v1.1.0) +
a per-APU-channel **mute mask** (`[audio] channel_mask`). GeraNES exposes per-channel
**volume** for the 5 APU channels *and* the expansion-audio chips
(`ref-proj/GeraNES/.../GeraNESApp.MenuUI.inl:400-402,1154-1175`). Add per-channel
gain (pulse1/pulse2/triangle/noise/DMC + VRC6 / VRC7-OPLL / MMC5 / Namco-163 /
Sunsoft-5B when present), surfaced in the Audio Settings tab with live application
through the existing lock-free audio-queue path. Default = all 1.0 → byte-identical
output. Channel discovery is dynamic (only show expansion sliders for the loaded
mapper). **Small/Medium**, frontend-only.

### D — Debugger / devtools finish (complete the GeraNES-class vision) → beta.2

Extends the existing CPU/PPU/APU/Memory/OAM/Mapper panels + Memory Compare (v1.3.0)
+ trace logger + event viewer + address breakpoints (v1.1.0). All behind
`debug-hooks` (output-only; determinism-neutral).

- **D1. Symbol-file loading** — load `.sym` / Mesen `.mlb` / FCEUX `.nl` label
  files and annotate the disassembler + breakpoint/trace views with function/label
  names. (Was in the v1.3.0 C plan but only Memory Compare actually landed in
  PR #84 — so this is genuinely open.) Ref:
  `ref-proj/GeraNES/.../GeraNESApp.h:100-105`. **Medium.**
- **D2. Event-driven breakpoints** — extend address breakpoints to break on NMI/IRQ
  entry/exit, sprite-0 hit, OAM/DMC DMA, and PPU/APU/mapper register read/write,
  each reporting frame/CPU-cycle/scanline/dot. Ref:
  `ref-proj/GeraNES/.../GeraNESApp.CpuBreakpointsWindowUI.inl:58-89`. **Medium.**
- **D3. HD-pack per-pixel inspector** — trace the HD-pack composition per pixel
  (matched CHR hash, replacement asset, palette index, condition/override reason).
  Complements v1.3.0 E1 (HD-pack conditions). Behind `hd-pack`. Ref:
  `ref-proj/GeraNES/.../GeraNESApp.ModPixelInspectorWindowUI.inl`. **Medium.**

### E — Browser / wasm QoL parity → beta.3

Bring the web build toward native QoL (browser RA itself is v1.3.0 I).

- **E1. wasm TAS movie `.rnm` I/O** — record + **download** and **upload** + replay
  `.rnm` movies in the browser (native already has record/play/branch; the wasm
  path lacks file I/O). Reuse the existing `wasm_io` base64/Blob/file-picker
  helpers. **Medium.**
- **E2. IndexedDB save-states** — move browser save-states/slots from `localStorage`
  (small, string-only) to IndexedDB for larger, binary, multi-slot persistence +
  the thumbnail grid the native Save-States manager has. **Medium.**

### F — Performance pass (measured, determinism-safe) → beta.3

~24% NTSC-frame headroom today (nestest 3.92 ms, flowing_palette 2.49 ms headless).
All items below touch CPU/PPU/mapper dispatch, DMA, or resampler infra — **zero
synthesis / pixel-emission changes** → determinism-safe. **Measure-first**: save a
`full_frame` Criterion baseline, land, re-baseline; re-run AccuracyCoin 139/139 +
visual golden after the set; re-run the PGO >3% promotion gate.

- **F1 (P1, HIGH/SMALL):** cache scanline-stable PPU rendering flags at dot 0
  (eliminate ~7 per-dot branches × 89,342 dots) + mark the hot pixel-fetch/
  shift-register helpers `#[inline]` (`crates/rustynes-ppu/src/ppu.rs`) + verify the
  v2.8.0 BLEP delta-ring auto-vectorization on ARM64/wasm. Expect ~2–5% headless.
- **F2 (P2, MED):** MMC5/ExRAM dispatch unification (cache an `exram_active` flag;
  bitflags instead of enum match) + BLEP phase-row caching in
  `crates/rustynes-apu/src/blip.rs`.
- **F3 (P3, LOW, batch):** reorder the `parse()` mapper match by frequency (hot 0/1/
  2/4/5 first) + gate the controller-strobe countdown + DMA get/put parity enum.

### G — Mapper breadth + BestEffort verification continuation → beta.4/rc (stretch)

- **G1.** Continue the BestEffort sweep toward more of the GeraNES ~258-mapper set
  (honesty-gated, ZERO-risk, register-decode + save-state tested only) — pick the
  next tranche of real-but-simple boards (target ~101 → ~115+).
- **G2.** Input-driven boot-smoke re-verification of the v1.3.0 D1 BestEffort
  "backdrop-only-without-input" cases (m94/107/111/143/177) to distinguish
  needs-input from a port bug — fix any genuine banking bugs found (like the m218
  16 KiB-PRG fix the no-input pass already caught). Commit screenshots only; ROMs
  stay gitignored in `tests/roms/external-besteffort/`.

### H — CLI / terminal help & argument UX → beta.2

Today the native binary hand-rolls argv parsing with a single `print_usage()` on
`-h`/`--help` (`crates/rustynes-frontend/src/main.rs:29-63`; the comment even says
"no need for clap in v0"). v1.4.0 modernizes the terminal CLI into the kind of rich,
interactive help advanced CLI tools ship. Native-only (a browser tab has no
terminal); the wasm `main` stays the empty shim. **Zero determinism surface**
(all pre-emulation); deps are frontend-only so the `no_std` chip stack is untouched.

- **H1. Adopt clap 4 + rich styled `--help`.** Replace the manual parser with a
  clap 4 `Command` (derive), preserving `rustynes <ROM>` + existing flags. clap
  styles `--help` with ANSI by default; set `Command::styles(Styles::styled())`
  (header / usage / literal / placeholder accents), wire version/author from
  `env!`, and add a colored "Examples" + "Keyboard" `after_help` via `color-print`'s
  `cstr!` → `StyledStr`. Auto-provides `-h`/`--help`/`-V`/`--version`. Honor
  `NO_COLOR` / `--color <auto|always|never>` (anstyle/anstream auto-detect TTY).
- **H2. `help` subcommand + topic help.** clap auto-generates `rustynes help` and
  `rustynes help <topic>` once subcommands exist. Topics: `controls`, `hotkeys`,
  `gamepad`, `features`, `mappers`, `config`, `scripting`, `netplay`, `about` —
  each a styled, sectioned page rendered from a single structured `help` module
  kept in sync with `docs/` + the keybinding tables (so CLI help and docs can't
  drift). `rustynes completions <bash|zsh|fish|pwsh>` via `clap_complete` (cheap,
  expected of polished CLIs).
- **H3. Interactive TUI help browser** (`rustynes help` with no topic on a TTY, or
  `rustynes help --interactive`). A full-screen **ratatui + crossterm** browser
  behind a default-on **`help-tui`** feature (a minimal build can drop it): a
  left topic tree / `Tabs` row, a scrollable colored content pane (`Paragraph` +
  `Scrollbar`), incremental search (`/`), keyboard nav (↑↓ / Tab / PgUp-Dn /
  Home-End), and quit (`q`/`Esc`). Reuse the H2 structured `help` module as the
  single content source. **Non-TTY detection** (piped / not a terminal) →
  gracefully fall back to the static styled H2 page, so `rustynes help mappers |
  less` still works and CI/headless never blocks on a TUI. (Helper crates from the
  research: `tui-tabs` for the bordered tab row, `ratatui-cheese`'s help widget,
  `ratatui-interact`'s TreeView/MenuBar — adopt only if they pull their weight;
  plain ratatui suffices.)
- **H4. Tests + constraints.** clap `command().debug_assert()` in a test; arg-parse
  unit tests (positional ROM, flags, bad-arg exit code 2); a non-TTY-fallback test;
  a help-topic-registry completeness test (every topic renders). **Gate the new
  native deps (clap, clap_complete, ratatui, crossterm, anstyle/color-print) out of
  the wasm target** so the wasm size budget is unaffected; keep them in the
  `[target.'cfg(not(target_arch = "wasm32"))'.dependencies]` table. Files: rewritten
  `main.rs`, new `cli.rs` (clap `Command` + topic registry) + `help_tui.rs`
  (ratatui browser, `help-tui`-gated). Update `docs/frontend.md` + `README` usage.

## Verification (every beta + finals)

- **Hard gate:** `cargo test --workspace --features test-roms` → AccuracyCoin
  **100% (139/139)** (+ the new committable suites), oracle byte-identical, nestest
  0-diff. A1/A2/A3 add ROMs to the gate; re-baseline only with them in.
- **Quality:** `cargo fmt --check`; `clippy -D warnings` per-feature (default +
  `scripting,hd-pack` + `retroachievements` + both wasm flavours — **never
  `--all-features`**); `RUSTDOCFLAGS=-D warnings cargo doc --workspace --no-deps`;
  `no_std` `rustynes-core` cross-compile (`thumbv7em-none-eabihf`);
  `pre-commit run markdownlint --all-files` (pinned v0.39.0).
- **Web:** wasm clippy + `trunk build` (both flavours) + `wasm_size_budget.sh`;
  keep `Trunk.toml` wasm-bindgen pin == `Cargo.lock`.
- **Perf:** PGO/BOLT promoted only on the >3% Criterion-stable bar;
  `full_frame` baseline before/after F.

## Out of scope / deferred to v2.0 (document, don't build)

The fractional-master-clock refactor (C1 IRQ-sample, `$2002`/`$2007` sub-cycle,
PAL 3.2:1 stale-shifter; ADR 0002); full Vs. DualSystem two-CPU/two-PPU emulation
(detection shipped v1.3.0 D2; no committable oracle — `docs/audit/
vs-dualsystem-design-2026-06-11.md`); FDS-proper (blocked on the precise scheduler);
mobile (iOS/Android); full Mesen HD-pack parity beyond conditions/bg-region; OAM +
open-bus DRAM decay (by-design); niche peripherals (microphone, Konami/Bandai Hyper
Shot, Barcode Battler, R.O.B., OEKA Kids, Pachinko); the aggressive mapper sweep
toward the full ~300 set beyond the BestEffort tranches.

## Maintainer decisions (confirm before scope-lock)

1. **Codename** — "Fidelity" (proposed) vs. another (e.g. "Keystone", "Burnish").
2. **A1 DMC-DMA** — confirm it's in (it's the one Medium accuracy item with real
   commercial-game payoff; ~Medium effort + test-ROM-gated). Recommended IN.
3. **D scope** — all three devtools (symbol/event-bp/HD-pixel) vs. a subset.
4. **G stretch** — include the mapper-breadth continuation in v1.4.0 or hold it.
5. **Theme weighting** — accuracy+test-hardening-first (A/B) vs. devtools/UX-first
   (C/D); the train above does accuracy first.
6. **H CLI help depth** — full ratatui interactive TUI browser (H3) vs. just clap 4
   styled `--help` + `help <topic>` subcommands (H1/H2). Recommended: do H1/H2
   unconditionally (low cost, big polish), gate the H3 TUI behind the default-on
   `help-tui` feature so it's optional. Also confirm adopting **clap 4** at all
   (it replaces the hand-rolled parser — clean + future-proof as flags grow, but
   it is a new native dependency).
