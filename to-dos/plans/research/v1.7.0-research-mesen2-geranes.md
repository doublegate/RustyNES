# RustyNES v1.7.0 — Research: mining Mesen2 + GeraNES (+ RA depth) for the next tier

> Generated 2026-06-17 (read-only research). Four parallel passes:
> (1) Mesen2 debugger + Lua source, (2) Mesen2 HD-pack/UI/cheats/audio source,
> (3) GeraNES source + RetroAchievements depth (grounded in RustyNES code), and
> (4) web research against the official `mesen.ca/docs` + `SourMesen/Mesen2`.
> Cross-referenced against RustyNES current state: `docs/STATUS.md`,
> `docs/frontend.md`, `docs/scripting.md`, `docs/cheevos-browser.md`,
> `docs/adr/0010..0015`, `CHANGELOG.md [Unreleased]`, and a direct read of the
> frontend/cheevos source.

## Positioning — what v1.7.0 must clear

v1.7.0 is the release **after** the in-development **v1.6.0 "Studio"** (MAXIMAL
scope, A–I, currently landing on `main`: TAStudio piano-roll, `.fm2`/`.bk2`
movie interop, Mesen2-class **expression/conditional breakpoints + watchpoints**,
**full hex editor with freeze**, **RAM watch/search**, watch window, **CDL**
output-only, off-axis accuracy, mappers → ~150 + **UNIF loader** + **JY ASIC**,
FDS-proper, **A/V recording**, **HD audio + spatial conditions + full
compositor**, **RetroArch shader import**) and **before** the **v2.0
fractional-master-clock refactor** (ADR 0002).

So the candidates here are the **next tier beyond v1.6.0**. The hard contract is
unchanged: determinism (same seed + ROM + input ⇒ bit-identical FB + audio),
**AccuracyCoin 139/139 + oracle byte-identical** after every change, additive /
off-by-default (shipped/native/`no_std`/wasm byte-identical where off), dev tooling
behind `debug-hooks`, scripting behind `scripting`, HD-pack behind `hd-pack`,
browser-RA behind `browser-cheevos`. Chip stack stays `#![no_std]`.

**Independently verified RustyNES gaps** (direct source read, not just inferred):

- PPU panel (`debugger/ppu_panel.rs`) is a **read-only** pattern/nametable viewer —
  no tile/CHR/palette/nametable **editor** with writeback.
- **No CallstackManager, no step-over/step-out/step-back, no profiler** anywhere
  in `crates/` (greps clean).
- **No localization / i18n** anywhere (no string table, no translate helper).
- Audio uses cpal `default_output_device()` only — **no device picker**; only a
  5-band EQ (`eq.rs`, NSF-scoped) + band-limiting filters; **no reverb / crossfeed /
  stereo panning**.
- Browser-RA is the documented **v1.5.0 carryover**: build track proven, casual-only
  structural, auth-proxy stub + contract, UI caveat — but trampoline marshalling,
  proxy deploy, and live verify remain.
- RustyNES's vendored rcheevos is **12.3.0** (newer than ref-proj 12.1.0) — no
  version chase; native RA event model is rich (events 1–19 modeled), but several
  events are **decoded-then-dropped** (genuine UI gaps below).

## Workstream R — Finish browser RetroAchievements (the named v1.7.0 carryover) + native RA depth

### R-FINISH. Complete browser RetroAchievements (ADR 0015 acceptance gate)

- **Source:** `docs/cheevos-browser.md` §"Status + remaining", `docs/adr/0015`,
  `crates/rustynes-frontend/src/wasm_cheevos.rs`, `web/cheevos/ra_glue.js`.
- **Checklist (≈ one L + two M + one S):**
  1. **Auth-proxy deploy (M, maintainer-manual)** — host `auth_proxy_stub.py` (or
     hardened equiv) with TLS + CORS locked to the Pages origin; coordinate the
     `RustyNES/<ver> rcheevos/<ver>` User-Agent + casual intent with the RA team;
     set `RA_PROXY_BASE` in `ra_glue.js`.
  2. **`ra_glue.js` rc_client trampoline marshalling (L)** — wire via Emscripten
     `addFunction`: a **read-memory** trampoline (NES CPU-bus byte from wasm), a
     **server-call** trampoline (`fetch(${RA_PROXY_BASE}/ra)` raw req/resp), an
     **event-handler** trampoline (serialize each `rc_client_event_t` to the

     the native HUD path (reuse, don't rebuild).
  4. **Live-browser verify with a real RA account (S, the acceptance gate)** — log
     in via the proxy, confirm a casual unlock toast; flips ADR 0015 to Implemented.
- **Why v1.7.0:** the only sanctioned carryover; closes the longest-standing
  documented deferral; native RA unaffected.
- **Effort:** L overall. **Gap:** yes (carryover). **Flag:** `browser-cheevos`
  (wasm-only, default-off). **NOT v2.0.**

## Workstream EDIT — Editing-capable debugger viewers (the standout Mesen2 differentiator)

Mesen2's distinguishing trait: its PPU/memory tools **edit and write back**. RustyNES
v1.6.0 brings Mesen2-class *inspection*; this is the *editing* tier. All behind
`debug-hooks`; every mutation is a deliberate dev poke gated **exactly like the
v1.6.0 hex-editor poke** (never on a save-state-replay / TAS / netplay path), so
shipped/oracle builds are byte-identical. **None are v2.0 items.**

- **Source:** Mesen2 `Core/Debugger/PpuTools.cpp:221` `SetTilePixel(...)` (bit-packs
  every TileFormat incl. NES 2bpp); UI `UI/Debugger/.../TileEditorViewModel.cs`.
- **What:** paint CHR tiles, see the change live. **Why:** turns RustyNES's
  read-only pattern viewer into a true editor — headline creator/romhack feature.
  **Effort: M.** Gap: **yes** (`ppu_panel.rs` is view-only). `debug-hooks`. NOT v2.0.

### E2. Palette editor writeback

  `DebugPaletteInfo{RawPalette[512], RgbPalette[512]}`.
- **What:** edit palette entries in place (write `$3F00` range). **Effort: S**
  (UI atop existing palette viewer + named-palette editor). `debug-hooks`. NOT v2.0.

### E3. Nametable / OAM editing + `disableSideEffects` + 200-entry undo

- **Source:** Mesen2 `Core/Debugger/MemoryDumper.cpp` `SetMemoryValue(...,

- **What:** structured editor for mapper#/submapper/PRG-CHR sizes/mirroring/battery/

  RustyNES's perpetual mapper-coverage work + the per-mapper screenshot pass.
  **Effort: M.** Gap: **yes**. `debug-hooks`. NOT v2.0.

### E5. Runtime 6502 inline assembler

- **Source:** Mesen2 `Core/Debugger/Base6502Assembler.cpp` (two-pass label-resolving,

## Workstream DBG — Debugger depth tier-2 (execution control + analysis)

Beyond v1.6.0 breakpoints/watchpoints/hex/RAM-search. All behind `debug-hooks`,
observation-only on the **current** PPU-dot scheduler — **none are v2.0 items**
(step-scanline/step-frame ride the existing dot lockstep).

### D1. MemoryAccessCounter + uninitialized-read detection (foundational)

- **Source:** Mesen2 `Core/Debugger/MemoryAccessCounter.h/.cpp` —
  `AddressCounters{Read/Write/ExecStamp (master-clock u64), Read/Write/ExecCounter}`;
  `ProcessMemoryRead` returns `{Normal, FirstUninitRead, UninitRead}` →
  **break-on-uninitialized-read**.
- **What:** the data backbone for the hex-editor access-heatmap (v1.6.0 plans
  *coloring* but not the counter+timestamp engine), the RAM-search "by access" sort,
  and Lua `getAccessCounters`. **Why:** foundational; unblocks D-Lua and an actual
  bug-finder (uninit-read). **Effort: M.** Gap: **yes** (extends planned coloring).
  `debug-hooks`. NOT v2.0.

### D2. CallstackManager + step-over / step-out / run-to-NMI / run-to-IRQ / step-scanline / step-frame

- **Source:** Mesen2 `Core/Debugger/CallstackManager.h/.cpp`; `StepType` enum
  (`Step, StepOut, StepOver, CpuCycleStep, PpuStep, PpuScanline, PpuFrame,
  SpecificScanline, RunToNmi, RunToIrq, StepBack`); `deque<StackFrameInfo>` (511,
  NMI/IRQ flags, return-addr/SP tracking).
- **What:** the *stepping verbs* + a call-stack panel (v1.6.0 has breakpoints but
  not over/out/run-to/step-scanline-frame). **Why:** large debugger-UX win.
  **Effort: L** (JSR/RTS detection + SP compare; threads the existing scheduler, not
  a clock refactor). Gap: **yes** (no callstack/step verbs in `crates/`). `debug-hooks`.
  **NOT v2.0** (rides existing dot scheduler).

### D3. Performance Profiler (per-function exclusive/inclusive cycles)

- **Source:** Mesen2 `Core/Debugger/Profiler.h/.cpp` —
  `ProfiledFunction{Exclusive/InclusiveCycles, CallCount, Min/MaxCycles}`, keyed off
  call-target, driven by the same JSR/RTS stack as D2 (co-build).
- **What:** see where the running ROM's 6502 cycles go. **Why:** completes the
  reverse-engineering/optimization toolset for homebrew authors. **Effort: M** (given
  D2). Gap: **yes** (no profiler). `debug-hooks`. NOT v2.0.

### D4. DisassemblySearch (find / find-all across banks) + GoToAll navigator

- **Source:** Mesen2 `Core/Debugger/DisassemblySearch.h/.cpp`
  (`SearchDisassembly`/`FindOccurrences`, MatchCase/WholeWord/Backwards, searches
  mnemonics/comments/labels/operands, wraps banks).
- **What:** search the disassembly. **Effort: M.** Gap: **yes**. `debug-hooks`. NOT v2.0.

### D5. Disassembly label + multi-line comment editing with per-ROM workspace

- **Source:** Mesen2 debugger docs + `DisassemblyInfo`; labels replace addresses
  everywhere, multi-line/inline comments, a Function List, persisted per-ROM.
- **What:** elevates the disassembler into a reverse-engineering workbench; pairs
  with RustyNES's existing `.sym`/`.mlb`/`.nl` loader (v1.4.0). **Effort: M–L** (label
  + comment store + workspace file format). Gap: **yes**. `debug-hooks`. NOT v2.0.

### D6. Editable Register Viewer (poke CPU/PPU/APU state while paused) + Set-Next-Statement

- **Source:** Mesen2 debugger "Emulation Status" panel — most fields editable while
  paused; "Set Next Statement" redirects PC.
- **What:** direct poke of register/state while paused. **Effort: M.** Gap: **yes**
  (panels are read-only). `debug-hooks`. NOT v2.0 (just register writes).

### D7. Trace logger advanced — in-memory addressable ring + PPU-dot columns + conditional filter

- **Source:** Mesen2 `Core/Debugger/BaseTraceLogger.h` — 30k-row circular buffer,
  `GetExecutionTrace(row, offset)`, `RowDataType` columns (incl. Scanline/HClock/
  EffectiveAddress/MemoryValue), `[Tag,width,h]` format parser, RPN condition filter.
- **What:** the addressable ring + dot/scanline/effective-address columns extend the

### D8. ExpressionEvaluator NES token-set parity (spec)

- **Source:** Mesen2 `Core/Debugger/ExpressionEvaluator.Nes.cpp` — tokens `a/x/y/sp/
  pc/ps`, flag bits, PPU `frame/cycle/scanline/v/t/sprite0hit/verticalblank/
  spriteoverflow`, `irq/nmi`, `[addr]`/`{addr}`, bp-context `value/address/isWrite/
  isRead/isDma/isDummy/romAddress`, full C operators.
- **What:** the *spec* for the token set v1.6.0's "ExpressionEvaluator-class"
  conditional breakpoints must expose to reach parity (unlocks watch window +
  conditional trace nearly for free). **Effort: M.** Gap: extends planned. `debug-hooks`.
  NOT v2.0.
