# v2.0.3 → v2.0.4 — The 2-cycle-ALE Fetch Refactor (ADR 0030 "Option 1")

> **Scope: a dedicated Timebase-scale campaign**, chosen by the maintainer (2026-07-08)
> after the bounded first-principles refine was proven infeasible at whole-dot resolution
> (ADR 0030 update). Goal: model each PPU VRAM access as a genuine **2-dot transaction** so
> the octal latch *naturally* carries the stale byte, making both AccuracyCoin PPU tests pass
> **at shipped default** (141/141) with a physically-correct model — no `+1 coarse-X`
> approximation. Mirrors the v2.0.0 "Timebase" beta-train ceremony (phased, flag-gated,
> full-battery-per-phase).

## Why this is Timebase-scale (the blast radius)

The BG fetch cadence is the calibration point for **sprite-zero-hit dots, MMC3 A12 IRQ timing
(`mmc3_test_2`), MMC5 scanline detection, BG shift-register reload, and the 60-ROM commercial
byte-identity oracle**. Splitting every fetch from 1 dot into a 2-dot (ALE / read) transaction
moves *when* each address is driven onto the bus, so a shifted A12 edge silently breaks an IRQ
test. This is exactly the risk ADR 0030 flagged; the phased battery gate is what makes it safe.

## Target model (from TriCNES `9199870`, `ref-proj/TriCNES/Emulator.cs`)

- **Even dot (ALE):** drive the full 14-bit fetch address onto `address_bus`; load
  `octal_latch = (address_bus & 0xFF)`. A12/mapper notify fires here (real bus event).
- **Odd dot (read):** effective address `(address_bus & 0x3F00) | octal_latch`; read VRAM/CHR;
  write the DATA byte back into the low bus (`address_bus = (address_bus & 0xFF00) | data`).
- **`$2006` 2nd write:** stage `CopyV` with a **3-dot delay** (`PPU_Update2006Delay`); apply `v`
  after the delay; on the next FetchVideoMemory `if CopyV { address_bus = v }` splices with the
  (now naturally stale, one-tile-ahead) `octal_latch`.
- **`$2007` read/write during render:** the ALE overlap freezes the latch on the DATA byte
  (`$0FFF`) — the ALE+Read path — reusing the existing `render_data_bus` machinery.

## Phases (each = a gated increment behind default-off `mc-ppu-2cycle-ale`)

- **Phase 1 — scaffold + STOP-OR-GO.** Introduce the 2-dot fetch structure behind the flag WITHOUT
  changing observable behavior yet (even-dot drives address + loads latch; odd-dot reads via the
  splice, which is transparent for a coherent fetch). Gate: **flag-off byte-identical**
  (AccuracyCoin 139/141, nestest 0-diff) AND **flag-on keeps the FULL battery green** (nestest,
  mmc3_test_2, sprite-zero, blargg/kevtris) — i.e. the split alone regresses nothing. If flag-on
  can't hold the battery, STOP and diagnose before adding corruption.
- **Phase 2 — A12 / cadence re-alignment.** Move the A12 notify + shift-reload + sprite-zero dot
  references onto the 2-dot cadence; re-gate mmc3_test_2 + sprite-zero + the visual golden corpus.
  **DONE (2026-07-08):** the NT fetch became a true two-dot ALE via a new phase-0 `ale_drive_nt`
  (plain address) + a disarm-and-co-locate correction at the read dot for the MMC5-split case (no
  non-mutating peek needed); A12/mapper notify stayed on the intended address. mmc3 18/18,
  ppu_sprites, mmc1_a12 all green flag-on.
- **Phase 3 — the two corruptions, naturally.** Wire the delayed-`CopyV` (Hybrid) + the
  `$2007`-ALE-overlap (ALE+Read) so the latch carries the correct stale byte with NO reconstruction.
  Gate: **flag-on 141/141 across BOTH test alignments** + full battery.
  **DONE (2026-07-08) — CONVERGED, fully natural.** `copy_v_delay = 4` (RustyNES's fixed alignment
  = TriCNES delay-4 case) makes the phase-0 NT ALE load the one-tile-ahead `$19` before `v = t`
  lands → NT read splices `$2F19` naturally (`W2006 dot182 → HYBRID $2F19 dot186`); the ALE+Read
  freeze reuses `ppudata_sm_countdown` → `SMLAND $FF dot228 → STALE $0FFF dot230`. **AccuracyCoin
  141/141 flag-on**, 139/141 flag-off (byte-identical). 60-ROM oracle 58/60 byte-identical; the 2
  changed titles (SMB3, Uchuu Keibitai SDF) are legitimate `$2006`-during-render single-tile
  (7-8px) shifts, MORE TriCNES-faithful — snapshots left at flag-off values, re-bless deferred to
  the Phase-4 promotion. See ADR 0030's final update section.
- **Phase 4 — perf + 60-ROM oracle.** `cargo bench` (≤2 ms/frame headless); the 60-ROM commercial
  byte-identity oracle flag-on (maintainer, local dumps) + broader `$2007`/`$2006`-during-render
  titles. Add a CI job asserting flag-on 141/141.
- **Promote (v2.0.4).** Flip `mc-ppu-2cycle-ale` to default-on → shipped 141/141; retire the v2.0.2
  `mc-ppu-bus-addr-hybrid` whole-dot stand-in; weigh ADR 0028 (byte-identity/save-state) at the flip;
  final release ceremony. Fold in the plan-mapped **Android RC** work (v2.0.4 window).

## Guardrails (every phase)

- Default-off flag; flag-off byte-identical to the shipped build throughout.
- Full battery per phase: 141 AccuracyCoin + nestest 0-diff + blargg/kevtris + mmc3_test_2 +
  sprite-zero + 60-ROM byte-identity + ≤2 ms/frame. A12 timing is the top silent-regression risk.
- Cross-diff against `ref-proj/TriCNES` per-dot behavior (the `octal_trace` tooling from v2.0.2).
- One cargo job per worktree (`flock`, sccache-safe `RUSTC_WRAPPER=` + `flock -o`).
- Multi-session by design — this is the v2.0.0-scale campaign, not a single sprint.
