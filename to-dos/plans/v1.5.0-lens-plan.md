# RustyNES — Plan: v1.5.0 "Lens" (proposed codename)

> Generated 2026-06-16 from research across `nesdev_wiki/`, `ref-proj/GeraNES/` +
> `Mesen2/`, `docs/`, `ref-docs/`, `to-dos/`, and the v1.2.0–v1.4.0 plans —
> accounting for everything v1.3.0 "Bedrock" + v1.4.0 "Fidelity" already consumed.
> Codename suggestion: v1.0 Cycle-Accurate → 1.1 Scriptable → 1.2 Curator →
> 1.3 Bedrock → 1.4 Fidelity → **1.5 "Lens"** (theme: *seeing into the machine* —
> visualization + insight + scriptability). Rename freely.

## Context — and an honest scope read

By the end of **v1.4.0 "Fidelity"** RustyNES has consumed nearly the entire
non-v2.0 backlog: AccuracyCoin 100% (139/139), 101 mapper families, the GeraNES-class
devtools (Memory Compare, symbol-file loading, event-driven breakpoints), per-channel
audio mixing, HD-pack `<condition>`/`<background>` rules, netplay desync diagnostics,
the clap-4 + ratatui CLI, the DMC-DMA / triangle / duty accuracy items, test-ROM
hardening, wasm `.rnm` + IndexedDB QoL, and a determinism-safe perf pass.

**Three research passes agree on the landscape:**
- **Accuracy is exhausted below the v2.0 axis.** The remaining residuals
  (`mmc3_test_2/4` #3, two `apu_reset`, full `$2002`/`$2007` sub-cycle, the PAL 3.2:1
  ratio, stale-shifter) all share ONE root cause — the **fractional
  12-master-clocks-per-CPU-cycle refactor** — which is genuinely **v2.0-scale**
  (ADR 0002 stop-condition, 15+ documented rollbacks). Do NOT open it in v1.5.0.
- **RustyNES now matches or exceeds GeraNES** in accuracy, mapper breadth, audio,
  event-debugging, and netplay diagnostics. GeraNES's only remaining edge is
  **debugger *visualization* + a handful of UX-polish windows**.
- The big-ticket remaining items (master-clock, full Vs. DualSystem dual-core,
  FDS-proper, mobile) are all **v2.0-scale**.

**Therefore v1.5.0 is a focused, specialized release — not a broad feature year.**
Theme: **insight + scriptability + creator tooling + polish** — finish the last
GeraNES-class *visualization* devtools, deepen the Lua scripting surface into a real
dev/TAS API, add the creator/speedrun tooling, and land the QoL-polish quick wins.
**The honest "next big thing" after v1.5.0 is the v2.0 master-clock refactor** — the
maintainer may legitimately choose to sequence v2.0 *instead of* v1.5.0; this plan is
the strong case for a polished consolidation release first.

## Hard contract (unchanged)
Determinism: same seed + ROM + input ⇒ bit-identical FB + audio; **AccuracyCoin
139/139 + oracle byte-identical** after every change. New features additive /
off-by-default (the shipped, wasm, `no_std` builds byte-identical where off); new
devtools behind `debug-hooks`/`hd-pack`. Chip stack stays `#![no_std]` + alloc; only
`rustynes-frontend` + `rustynes-cheevos` carry `unsafe`. The core test harness never
consults a frontend DB.

## STANDING RESEARCH DIRECTIVE (carry through execution)
Research is **continual**: pin behavior against `nesdev_wiki/` + the committed test
ROMs; consult **GeraNES** (UI/UX) + **Mesen2** (accuracy) in `ref-proj/`; and **fold
in beneficial enhancements + determinism-safe perf wins opportunistically** even when
not listed here, respecting the hard contract (measured for perf, test-pinned for
accuracy). Surface anything larger-than-a-quick-win as its own ticket.

## Workstreams

### A — Debugger visualization (finish the GeraNES-class "Lens") → beta.1
All behind `debug-hooks`/`hd-pack`; output-only; determinism-neutral. These are the
genuine remaining GeraNES-edge items.
- **A1. Input Miniatures overlay** — a live on-screen panel drawing every connected
  input device (standard pad, Zapper, Arkanoid Vaus, SNES mouse, Power Pad, Family
  BASIC keyboard, multitap/Four Score) with real-time button/axis feedback. Ref:
  `ref-proj/GeraNES/.../GeraNESApp.InputMiniaturesOverlayUI.inl`. **Medium.**
- **A2. Graphical PPU Event Viewer** — a 341×312 per-dot read/write heatmap
  (blue=read, red=write) with hover/click cycle metadata, extending the existing
  event viewer + event-breakpoints. Ref:
  `ref-proj/GeraNES/.../GeraNESApp.EventViewerWindowUI.inl`. **Medium.**
- **A3. PPU scanline-trace viewer** — the existing nametable/CHR/palette panel + a
  per-scanline state-capture trace + CHR→PNG export. Ref:
  `ref-proj/GeraNES/.../GeraNESApp.PpuViewerWindowUI.inl`. **Medium.**
- **A4. HD-pack per-pixel inspector** (the v1.4.0 D3 carryover; behind `hd-pack`) —
  per-pixel composition trace (matched CHR hash, replacement asset, palette index,
  the `<condition>`/`<background>` that gated it — ADR 0014), with a blend slider.
  Ref: `ref-proj/GeraNES/.../GeraNESApp.ModPixelInspectorWindowUI.inl`. **Medium.**

### B — Scriptability depth (Lua dev/TAS API) → beta.2
Deepen the v1.1.0 Lua engine (`rustynes-script`, native; `onNmi`/`onIrq`/`setInput`
exist) into a real automation surface. Behind the existing `scripting` feature;
state-mutating calls gated exactly like `emu.write`/`setInput` (no-op in
netplay/TAS-replay/RA-hardcore — preserves determinism).
- **B1.** Memory API: `memory:peek/poke/read_range/write_range` (CPU + PPU spaces).
- **B2.** Cart/system queries: `cart:mapper_id()/prg_size()/chr_size()/sha256()`,
  region, frame counter.
- **B3.** Save-state scripting: `emu:save_state()/load_state()` (in-memory slots).
- **B4.** Debug hooks for scripts: `emu:on_breakpoint`, `emu:pause_at_frame`, and
  **symbol-label queries** pairing with the v1.4.0 symbol loading (`sym:addr("main")`).
- **B5.** A bundled example-script set (memory scanner, TAS frame analysis,
  game-state tracker) + `docs/scripting.md` expansion + API tests.

### C — Creator / TAS / speedrun tooling → beta.2
- **C1. TASVideos compatibility pass** — audit RustyNES against the TASVideos /
  extended emulator-test set beyond the 139 AccuracyCoin; wire any committable ones;
  fix genuine bugs found (pin first). Report which (if any) need the v2.0 axis.
- **C2. Replay/TAS window polish** — device-topology display + timebase in the movie
  UI (per GeraNES `ReplayWindowUI`); branch/seek UX.
- **C3. NSF waveform visualizer** — a per-channel scope for the NSF player (frontend
  eye-candy over the existing NSF/EQ).

### D — UX polish (the GeraNES QoL quick wins) → beta.3
Frontend-only; determinism-neutral. Mostly small.
- **D1. Full palette editor** — load/save/edit *named* custom palettes + per-index
  color picker (extends the existing `.pal` loader + palette viewer).
- **D2. Overscan WYSIWYG editor** — per-side (T/R/B/L) sliders with live preview +
  reset (replaces the binary hide-overscan toggle's discoverability gap).
- **D3. "Enhancements" grouped settings** — consolidate sprite-limit-disable /
  optional overclock / max-rewind-seconds into one Settings group (GeraNES
  `ImprovementsWindowUI`); each off-by-default. (Overclock + no-sprite-limit are
  non-accuracy *enhancement* modes — clearly labelled, never on by default, never in
  the determinism oracle.)
- **D4. Device-config windows** — Arkanoid Vaus + SNES-mouse sensitivity, Power Pad
  layout variants.

### E — Accessibility → beta.3
Frontend-only. Configurable UI scaling (egui zoom), colorblind-safe + high-contrast
themes (extend the light/dark/system theming), and keyboard-only menu navigation.
Low-risk, broadens reach; no existing user issues but a genuine quality bar.

### F — Mapper breadth continuation → beta.4 (stretch)
Continue the honesty-gated BestEffort sweep **101 → ~120** (register-decode +
save-state tested, never accuracy-gated) + promote any board to Curated where a
redistributable fixture surfaces + continue the input-driven boot-smoke re-verify of
the prior backdrop-only cases. ZERO determinism risk (the honesty gate holds).

### G — Browser RetroAchievements (the ADR-0015 carryover) → beta.4 / rc — MAINTAINER DECISION
The casual-mode-only browser RA from ADR 0015: an **Emscripten** (or pure-Rust)
rcheevos→wasm build track + an auth proxy for the browser-forbidden `User-Agent` +
casual-only gating (hardcore disabled) + loud in-UI caveats. **~2–3 wk + an external
dependency** (the emscripten toolchain is not in the build env; live-browser + RA-account
verification has no headless path — a maintainer-manual step like v1.2.0 F1/F3).
**This is the one larger item with an external/manual component** — include only if
the maintainer commits to standing up the toolchain; otherwise it remains the documented
ADR-0015 carryover. Native RA is unaffected either way.

### H — Frontend pacing & audio-sync performance → beta.3 (added 2026-06-16 from a real perf-log capture)
Source: `perf-logs/perf-Super_Mario_Bros_nes-20260616-231215.csv` (v1.4.0 release,
143.975 Hz display, `auto`→`wallclock` pacing, Mailbox, run-ahead=1, rewind on).
**Baseline is reassuring** — raw frame `cost_mean ≈ 8.5 ms` / `p99 ≈ 9.2 ms` /
`max ≈ 10.2 ms` vs the 16.639 ms budget (~51%, tight). So the v1.4.0 perf pass
holds and the **core synthesis is NOT the bottleneck**; every issue is in the
**frontend pacing/present/audio layer** (determinism-safe — pacing lives in the
frontend by contract). Measured problems: (a) recurring **produce stalls**
`produced_max` 50–128 ms with `catchup_bursts` 9→62 + `snap_forwards` 3→12 while
cost stays flat (⇒ blocking/scheduling, not compute); (b) a **cyclic
degrade/recover ~10–15 s beat** between 60.0988 Hz content and 143.975 Hz display;
(c) **audio underruns** 3→12 over 58 s + `audio_queued_ms` 68–91 ms oscillating
around the 60 ms target; (d) **produced_dropped** 67→374 wasted frames; (e)
**no GPU timing** (`gpu_ms = "-"`). Corrections (all `rustynes-frontend` except a
*bit-identical* allocation-only core tweak; measure-first, re-verify AccuracyCoin):
- **H1. Lock-free framebuffer handoff** — replace the "copy FB under a brief emu
  lock" present path with a triple-buffer / SPSC swap so the 144 Hz present thread
  never blocks the emu thread. `perf record` first to confirm lock contention is a
  stall source.
- **H2. Hybrid sleep+spin wallclock pacer** — sleep-until-near-deadline then short
  busy-spin (or a precise timer) to kill the 50–128 ms OS-scheduler jitter spikes;
  cap catch-up-burst length so one stall can't snowball into a snap-forward.
- **H3. Allocation-free run-ahead/rewind snapshot** — audit `EmuCore::snapshot` +
  the rewind ring for per-frame `Vec` allocation; reuse preallocated buffers (a
  snapshot ring). Core-adjacent: **must stay bit-identical** (allocation strategy
  only) — re-run AccuracyCoin + determinism.
- **H4. Audio DRC + buffer tuning** — retune the DRC setpoint/gains so the queue
  tracks 60 ms instead of oscillating 68–91 ms; widen the resampler correction
  range; add a larger low-watermark / one-time adaptive buffer bump on high-refresh
  displays. Goal: **0 underruns**.
- **H5. GPU timestamp queries → `gpu_ms`** — wire wgpu timestamp queries so the
  perf log reports real GPU/present cost (currently blind); surface `presented_max`
  present-gaps in the Performance panel anomaly readout.
- **H6. High-refresh pacing polish** — for N×60 Hz panels (120/144/240) consider a
  present-aligned-to-production cadence under Mailbox to smooth the 60-on-144 beat;
  reuse the v1.3.0 B3 beat-vs-judder diagnostic to keep the distinction visible.
- **H7. Perf-log regression gate** — script an SMB perf capture into the bench/CI
  flow so `produced_max` / `underruns` / `catchup_bursts` become tracked signals,
  not one-offs.
- **H8. Perf-monitor → CSV-log parity (the exporter has drifted behind the panel).**
  The CSV writer (`crates/rustynes-frontend/src/perf_log.rs`, header at L206–208 /
  value row L232–241) no longer emits everything the Performance Monitor panel
  (`debugger/perf_panel.rs`) computes from `PerfSnapshot` (`perf.rs`). Confirmed
  gaps: `gpu_ms` is a column but is always empty (`-` — populated by **H5**);
  `present_mode_fell_back` is tracked but not logged; and the panel's derived
  readouts (run-ahead / rewind state, the audio DRC ratio + resampler rate, the
  present/produce **beat** + pacer-anomaly counters) have no columns. Bring the
  exporter to full parity with the live panel/`PerfSnapshot`, add the header-block
  fields the panel knows but the log omits, and **add a test asserting every
  panel-surfaced metric has a CSV column** so the two can't silently drift again.
This is real user-facing value **independent of accuracy**, so it also strengthens
the case for shipping v1.5.0 before the v2.0 axis (decision #2).

### I — Native-UI fixes + menu overhaul + in-app Documentation → beta.3 (maintainer hands-on report, 2026-06-17)
Concrete bugs + UX enhancements from real native-app testing. All frontend-only,
determinism-neutral. Research `ref-proj/` (GeraNES / Mesen2) + the web for each
and over-deliver on polish/optimization beyond the literal list.
- **I1. BUG — "File → Copy Screenshot to Clipboard" no-ops.** Status bar says it
  copied but nothing reaches the system clipboard. Wire a real clipboard image
  write (e.g. `arboard` `set_image` with RGBA + width/height), keep the existing
  file-write path; verify on Linux (Wayland/X11). Fix the false success toast.
- **I2. BUG — Frame Advance + Fast Forward dead.** `\` (Frame Advance) does
  nothing (click or key); Fast Forward (`Tab`-hold) never engages + the menu item
  stays greyed; holding `Tab` just cycles the egui menu bar. Root cause likely
  egui consuming `Tab`/`\` before the hotkey handler (focus/`consume_key`), or the
  `MenuAction` enabled-predicate for FF being wrong, or the hotkeys not dispatched
  when no widget has focus. Re-route these through the global hotkey path (like
  the working F-keys), pick a FF key that egui doesn't eat (or suppress egui's
  Tab-navigation while the emu is focused), and fix the FF contextual-enable
  predicate. Add a smoke test for the hotkey→MenuAction mapping.
- **I3. UX — Settings pane.** Shaders tab: the "Shader stack (composable)"
  collapsing header should default-OPEN (it currently needs a manual click).
  Input tab: there's a "Save to disk" button although settings already auto-save
  on change — either remove it (rely on the per-setting auto-save from v1.3.0) or
  relabel it to its real function (e.g. "Export config…"); first CONFIRM auto-save
  actually fires for every Input-tab control and fix any that don't.
- **I4. ENHANCEMENT — Cheats pane game-DB picklist.** Surface selectable known
  cheats for the loaded ROM from the v1.2.0 Game-Genie code DB (`genie_database.tsv`,
  CRC32/sha256-keyed) as a pick-list that feeds the existing `genie.rs` decode +
  `cheats.rs` persistence — alongside the existing manual-code entry + raw-RAM
  cheats. (Consider widening the bundled DB.)
- **I5. ENHANCEMENT — Input Display colors.** Keep D-pad **green**; make
  Select/Start **yellow**; make B/A **Nintendo red** (`#E60012`-ish). (Mirror the
  same palette in the A1 Input Miniatures overlay for consistency.)
- **I6. BUG — "Tools → ROM Database" won't open standalone.** It only renders
  while another `nes`-reading tool panel (e.g. Cheats) is open, and closing that
  hides ROM DB too. Cause: the hidden-render "locked branch" (which copies the FB
  under a brief lock + renders with `nes=None`) is only taken when the overlay or a
  known `nes`-reading panel is open — ROM DB isn't in that predicate. Add ROM DB
  (and audit every other `nes`-reading tool panel) to the locked-branch trigger so
  it opens/persists independently.
- **I7. REFACTOR — retire "Debug → Show Debugger" (`` ` ``).** The toggleable
  CPU/PPU/APU/mem overlay is largely redundant now (the dedicated tool panels +
  beta.1 viz cover it). Remove/deprecate the ` overlay; relocate its only
  still-useful readout — **RetroAchievements** status — into the bottom status bar
  between the emulator-state label (e.g. "Running") and the `FPS:` counter. Keep
  the individual debugger tool panels (they're separate). (Confirm with the
  maintainer if fully removing vs hiding the ` overlay.)
- **I8. ENHANCEMENT — Mapper panel depth.** The current pane is sparse. Extend
  with: mapper id+submapper + board/ASIC name + tier (Core/Curated/BestEffort),
  PRG/CHR ROM+RAM sizes + bank counts, current bank mapping per CPU/PPU window
  (live `$8000-$FFFF` / `$0000-$1FFF` bank indices), mirroring (+override source),
  IRQ type/state (MMC3 A12 counter, MMC5 scanline, VRC/FME-7/N163 cpu-cycle) when
  present, expansion-audio chip, battery/NVRAM, register last-write log. Research
  GeraNES `MapperInfo`/Mesen2 mapper-info panels for the canonical field set.
- **I9. ENHANCEMENT — Help → Keyboard Shortcuts.** Add a separator between the
  emulator hotkeys and the controller mapping; add a Player selector (P1 / P2 /
  Power Pad / Family BASIC kbd / …) showing each device's key bindings — reflecting
  the **live custom bindings** from the `[input]`/`[shortcuts]` config (v1.3.0
  remappable shortcuts), not just defaults.
- **I10. NEW — Help → Documentation pane.** An in-app docs browser (reuse the
  `rustynes help` topic registry from v1.4.0 H so CLI + GUI share one source):
  controls/hotkeys/gamepad/features/mappers/config/scripting/netplay + EXTENDED
  GUI-specific docs, an **About** section (version, license, build, links), and a
  **selectable per-release summarized CHANGELOG** (current + past). Searchable
  (`/`-style filter), logically grouped, egui-native (not the ratatui TUI — that's
  terminal-only). Author whatever content sections are missing. Consider a
  first-run "What's new in v1.5.0" surfaced from this pane.
- **Plus (over-deliver):** sweep ALL menus for dead/greyed/mislabeled items +
  contextual-enable correctness (the I2 + I6 class of bug may recur elsewhere),
  consistent iconography/tooltips, and any GeraNES/Mesen2 UX wins worth adopting.

I + D together are the beta.3 UX scope (with E accessibility + H perf). Implement
AFTER beta.2 (#102/#103) merges, off the updated main, to avoid frontend-file
conflicts. I1/I2/I6 are genuine BUGS — prioritize. Each behind the same

## Verification (every beta + finals)
`cargo test --workspace --features test-roms` → **AccuracyCoin 139/139** + oracle
byte-identical + nestest 0-diff; `fmt --check`; `clippy -D warnings` per-feature
(default + `scripting,hd-pack` + `retroachievements` + both wasm flavours — never
`--all-features`); `RUSTDOCFLAGS=-D warnings cargo doc --workspace --no-deps`; `no_std`
cross (`thumbv7em`); markdownlint v0.39.0; wasm `trunk` + size budget; `full_frame`
bench if any hot-path touched.

## Out of scope / deferred to v2.0 (document, don't build)
The **fractional-master-clock refactor** (the C1 IRQ-sample / `$2002`/`$2007` sub-cycle
/ PAL 3.2:1 / stale-shifter residuals — ADR 0002); **full Vs. DualSystem dual-core**
(detection shipped; no committable oracle); **FDS-proper** (blocked on the precise
scheduler); **mobile** (iOS/Android); **full Mesen HD-pack parity** (neighbor
predicates / HD audio — land A4 first); OAM/open-bus DRAM decay (by-design); the
aggressive mapper sweep toward ~300.
