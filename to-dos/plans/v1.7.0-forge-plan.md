# RustyNES — Plan: v1.7.0 "Forge"

> **Scope LOCKED by the maintainer (2026-06-17):** **MAXIMAL — full A–H.** Every workstream tier the
> three research passes surfaced is in-scope. Codename **"Forge"** (the release where the tools become
> *writable* and *programmable*) — maintainer renames freely.

## Context

RustyNES shipped **v1.0.0 → v1.1.0 "Scriptable" → v1.2.0 "Curator" → v1.3.0 "Bedrock" → v1.4.0 "Fidelity"
→ v1.4.1 → v1.5.0 "Lens"**, with **v1.6.0 "Studio"** (the MAXIMAL TAS-authoring + debugger-depth +
off-axis-accuracy + breadth cut) in development. v1.6.0 already absorbs the *first* tier of reference-emulator
tooling: a TAStudio-class piano-roll editor (greenzone + lag log + markers + branches + `.rnmproj`), `.fm2`/`.bk2`
movie interop, Lua `frameadvance`/coroutine driving + memory domains/sized reads/`joypad`/`movie.*`, Mesen2-class
conditional breakpoints/watchpoints + a hex editor + RAM Watch/Search, A/V recording, the FCEUX Code/Data Logger,
FDS-proper, mappers→~150 + UNIF, HD-audio, and shaders.

**v1.7.0 "Forge" is the MAXIMAL *next tier*** — the layer a serious creator/TAS/hacker expects once the v1.6.0
spine exists: making the inspect-only tools **editing-capable**, making the TAStudio editor **programmable**,
giving the debugger **source-level/step/callstack** depth, scaling the rewind engine, opening a **host-mediated
IPC/automation** surface, hardening **dot/CPU-cycle-granular accuracy** (everything reachable *without* the v2.0
timebase rewrite), widening **mapper + expansion-audio + movie-format** breadth, and finishing the **browser
RetroAchievements** carryover. It is explicitly **NOT** the v2.0 master-clock/timebase rewrite (that is its own
major — see *Out of scope*).

> **Engine-lineage versioning note.** Deep technical docs reference an upstream "v2.0" master-clock program —
> that already shipped as the v1.0.0 cycle-accurate core. The *future* RustyNES **v2.0.0** is the separate
> one-clock + every-cycle-bus-access timebase refactor (ADR 0002). Everything in this plan rides the **current**
> dot-lockstep scheduler.

## Hard contract (unchanged — applies to every item below)

- **Determinism:** same seed + ROM + input ⇒ **bit-identical** framebuffer + audio. **AccuracyCoin 100%
  (139/139)** + the commercial-ROM oracles **byte-identical** must hold after every change. nestest 0-diff.
- **Off-by-default:** every new feature is additive and feature-gated so the **shipped / wasm / `no_std`** builds
  stay **byte-identical** to v1.6.0. Prove byte-identical with the relevant feature **on and off**.
- **Chip stack stays `#![no_std]` + alloc;** only `rustynes-frontend` + `rustynes-cheevos` carry `unsafe`.
- **State-mutating paths gate exactly like `emu.write`** (no-op under netplay / TAS replay / RA-hardcore) — this
  now covers the new editing-tool writeback, `tastudio.*` mutators, Lua `setState`/value-modifying callbacks, and
  the IPC input bridge.
- **Core test harness never reads any frontend DB/override.** A CI assertion guards it.
- **Accuracy is test-ROM-is-spec:** pin the failing test-ROM expectation **first**, then implement until it
  passes. Hot paths: no allocation, fixed arrays, profile (`cargo bench` + `perf`) before abstracting; ≤ 2 ms/frame
  headless.

> **STANDING DIRECTIVE — continual research + opportunistic improvement.** When implementing *any* workstream
> below, re-mine `nesdev_wiki/`, `ref-proj/` (BizHawk, Mesen2, GeraNES, ares, higan, FCEUX, puNES, TriCNES), and
> the web as needed to get the behavior right — and **make any beneficial enhancement or measured performance
> optimization you encounter en route, even if it is not explicitly listed here**, provided it respects the hard
> contract (byte-identical with the feature off; AccuracyCoin 139/139; pin-test-ROM-first for accuracy). The
> research dives in [`research/`](research/) are the backing detail for every item.

## Workstreams

### A — Editing-capable tools (the read-only → writable leap)

RustyNES's `ppu_panel.rs` and memory views are **verified read-only**; Mesen2's are not. This turns the inspect-only
debugger into a creator/RE **workbench**. All writeback is `debug-hooks`-gated and obeys the `emu.write` gate.

- **A1. Tile / CHR + palette + nametable + OAM editors (writeback)** — **L.** Edit CHR bytes, palette entries,
  nametable tiles/attributes, and OAM from the debugger panels; writes go through the same gated post-frame poke
  path as cheats. `crates/rustynes-frontend/src/debugger/{ppu_panel,mem_panel}.rs` + a small gated core write hook.
  *Determinism:* writes gated like `emu.write`; output-only when locked. NOT v2.0.
- **A2. iNES / NES 2.0 header editor + read-only "Cartridge Info" pane** — **M** (the read-only pane is an **S**
  subset, ship first). Inspect/edit the 16-byte iNES + NES 2.0 header *of a ROM file on disk* (mapper, mirroring,
  PRG/CHR sizes, battery, region, submapper, RAM sizes, console type). Source: FCEUX `iNesHeaderEditor.cpp`. Pairs
  with the v1.6.0 UNIF loader + mapper DB. *Edits a file, not the running core; read-only by default.* NOT v2.0.
- **A3. Inline 6502 assembler** — **M.** Assemble a line/region at an address into the gated write path (Mesen2's
  debugger assembler; ares parity). `crates/rustynes-frontend/src/debugger/cpu_panel.rs`. NOT v2.0.

### B — Scriptable TAStudio + full Lua API parity

v1.6.0 builds the *editor*; this makes it **programmable** (bots, generated TASes, analysis canvases).

- **B1. `tastudio.*` Lua control API** — **M.** `engaged`/`get|set|togglerecording`, `setplayback(frame|marker)`,
  `getseekframe`/`getselection`, `islag`/`setlag`, `hasstate`, marker get/set/remove, `submitinputchange`/
  `applyinputchanges` (queued atomic edit), `getbranches`/`getbranchinput`/`loadbranch`/`setbranchtext`. Wires the
  v1.6.0 input-log/branch/marker model to Lua. Source: BizHawk `TAStudioLuaLibrary.cs`. *Mutators gated like
  `emu.write`; behind `scripting`.* NOT v2.0.
- **B2. `tastudio` custom-column / per-cell draw + greenzone/branch callbacks** — **M.** `onqueryitembg|text|icon`
  + `clearIconCache`; `ongreenzoneinvalidated(fn)`, `onbranchload(fn)`. Programmable analysis canvas over the
  piano-roll grid. Overlay-only. NOT v2.0.
- **B3. Full Lua API parity** — **M.** `getScreenBuffer`/`setScreenBuffer`, the full `addEventCallback` enum,
  **value-modifying** memory callbacks, `getState`/`setState` (in-memory), `takeScreenshot`, `getScriptDataFolder`.
  Source: Mesen2 Lua API. *Mutators (`setScreenBuffer`/`setState`/modifying callbacks) gate like `emu.write`.* NOT v2.0.

### C — Debugger depth (source-level / step / callstack)

The biggest debugger-UX leap, all on the **current** dot scheduler (confirmed NOT v2.0).

- **C1. CallstackManager + step over/out + run-to-NMI/IRQ + step-scanline/step-frame** — **L.** No callstack/step
  exists in `crates/` today. Tracks JSR/RTS/IRQ/NMI/RTI for a live call stack; the step modes ride the existing
  `debug-hooks` exec log. Source: Mesen2 `CallstackManager`/step logic. *Output-only; `debug-hooks`.* NOT v2.0.
- **C2. MemoryAccessCounter + uninitialized-read detection** — **M.** Per-address read/write/exec counts +
  last-access stamps + an `UninitRead` flag (Mesen2 `MemoryAccessCounter`). Backbone for the hex heatmap + Lua
  access counters. *Output-only side-array; `debug-hooks`; byte-identical off.* NOT v2.0.
- **C3. ca65/cc65 `.dbg` source-line mapping** — **L.** The #1 remaining devtools gap vs Mesen2 (RustyNES has
  symbol *names* only). Parse the cc65 `.dbg` file → annotate the disassembly + memory views with source lines.
  Frontend parser. NOT v2.0.

### D — Timeline + scaling rewind engine

- **D1. HistoryViewer** — **M.** A scrubbable full-session timeline (Mesen2 spins a 2nd emu over the rewind ring)
  with **export-last-N-seconds-as-`.rnm`**. Reuses the v1.5.0 keyframe cache; strong synergy with v1.6.0 TAStudio
  and the live-netplay scrubber carryover. NOT v2.0.
- **D2. Zwinder-class compressed tiered rewind/greenzone** — **L, core-adjacent, determinism-critical.** XOR-delta
  + zlib over the v1.6.0 uncompressed greenzone, with density tiers (current/recent/ancient) + reserved anchors, so
  it scales to feature-length TASes. Source: BizHawk `ZwinderBuffer`/`ZwinderStateManager`. *Gate with a
  save→compress→decompress→restore **lossless round-trip equality** test — the restored state MUST equal the saved
  state.* No timebase change. NOT v2.0.

### E — Host IPC / automation (RustyNES as a platform)

The power-user tier that turns RustyNES into a platform for external bots / RL agents / randomizers / stream tools.
RustyNES's determinism is a *selling point* for reproducible RL episodes.

- **E1. Host-mediated `comm.*` IPC** — **L.** TCP socket + HTTP + WebSocket + memory-mapped-file bridge
  (`socketServerSend`, `httpGet/Post`, `ws_*`, `mmf*`). The script **never** gets raw sockets — the **host owns the
  connection** so the Lua sandbox guarantee (no `io`/`os`/`package`/net) is preserved. Source: BizHawk
  `CommLuaLibrary.cs`. *NEW non-deterministic input source → behind a new off-by-default `script-ipc` feature,
  host-mediated, and **disabled in replay/netplay/RA** exactly like `emu.write`. The core synthesis never sees it.*
  NOT v2.0.
- **E2. `client.*` automation verbs** — **M.** Scripted host control: open tools, `screenshot`/`screenshottoclipboard`,
  `setwindowsize`, `speedmode`/`frameskip`, `reboot_core`, `pause_av`/`unpause_av`, `addcheat`/`removecheat`. Glue for
  scripted reproducible capture pipelines (pairs with v1.6.0 A/V + E1). Source: BizHawk `ClientLuaLibrary.cs`. NOT v2.0.
- **E3. `userdata.*` (+ optional SQLite) persistence** — **S** (`userdata`) / optional **M** (SQLite). Per-script
  persisted KV store across runs (randomizer tables, bot logs). `userdata` may persist into save-states. NOT v2.0.

### F — Accuracy hardening (dot/CPU-cycle-granular, NOT v2.0)

Accuracy below the v2.0 timebase axis is **near-exhausted** — all four dives confirmed it. Each item is reachable
on the current scheduler and **pinned to a committed test ROM first**.

- **F1. Test-ROM hardening bundle** — **S each** (all ROMs already committed, 0 harness refs today): `ppu_read_buffer`,
  `vbl_nmi_timing` (**ignore** the `nmi_suppression` sub-ROM → that is v2.0), `sprdma_and_dmc_dma` + `dmc_tests`,
  `cpu_exec_space`, wider `read_joy3`, `volume_tests` (a real gap — relative-loudness, which kevtris omits),
  holy-mapperel **M28/M118/M180** + a **battery-save round-trip** oracle (none today), `scanline.unif`. Cheapest,
  highest regression value. *Wire into `rustynes-test-harness`; pin-first.*
- **F2. The two genuinely-new sub-v2.0 behaviors** — **M.** (a) APU **length-counter halt/reload race**;
  (b) **DMC load-DMA even/odd-cycle delay**. Pin each to a test ROM. Everything else ares flagged
  (pulse phase reset, sweep negation/mute, OAMADDR/OAM masks) is **verify-only** — already implemented; add an audit
  test, don't reimplement. NOT v2.0.
- **F3. PPU extra-scanlines overclock (determinism-gated)** — **M.** Scanline *insertion* at the existing dot
  resolution (Mesen2 `UpdateTimings`). **In-scope but hard-gated:** off by default, and **DISTINCT from the
  CPU-multiplier overclock, which IS a v2.0 item** (needs the timebase rewrite). Prove byte-identical at the default
  (zero extra scanlines).

### G — Breadth (mappers · expansion audio · movie formats · HD-pack authoring)

- **G1. Next reusable-ASIC BMC/pirate cores** — **M–L.** FK23C / COOLBOY / MINDKIDS / Sachen / Waixing / Kaiser
  clusters → ~**170–185 families** (+200–500 titles). **BestEffort / honesty-gated** (zero determinism risk, off the
  oracle). *Reconcile the exact mapper numbers against the v1.6.0-landed set first.* `crates/rustynes-mappers/src/sprintN.rs`.
- **G2. NSF expansion-audio synthesis** — **M, near-free.** Route the parsed NSF expansion bitfield's register writes
  into the **existing** VRC7/VRC6/FDS/MMC5/N163/5B synth cores — unlocks the bulk of the NSF library. *Improve the
  MIT `emu2413` VRC7 core if needed — do **NOT** adopt **Nuked-OPLL** (GPL/LGPL, incompatible with MIT-OR-Apache).*
- **G3. MMC5 expansion audio** — **M.** The one expansion chip started-but-deferred; documented gap. Core synth.
- **G4. Broad movie-format import** — **M.** `.fcm` (legacy FCEUX), Famtasia `.fmv`, VirtuaNES, Mednafen `.mc2` —
  the historical pre-`.fm2` TASVideos corpus → "play any NES TAS." Plus harden `.fm2`/`.bk2` export (rerecord count,
  MD5/SHA hashing). Same canonical fixed-power-on-seed alignment as v1.6.0 `.fm2`. Frontend readers. NOT v2.0.
- **G5. HD-Pack Builder** — **XL.** An in-emulator recorder → `hires.txt` + tile sheets (Mesen2). v1.6.0 *plays*
  HD-packs; this *authors* them. Output-only over the existing PPU tile-source export. Determinism-safe. NOT v2.0.

### H — Reach, polish, audio, RA, web, perf

- **H1. Finish browser RetroAchievements (ADR 0015)** — **L, code + manual.** The v1.5.0 emcc rcheevos→wasm side
  module + casual-only gating + auth-proxy stub exist. **Remaining:** the `ra_glue.js` wasm trampoline marshalling
  (`addFunction`), the auth-proxy **deploy**, and a **live-browser verify with a real RA account**. `browser-cheevos`
  off-by-default; RA is observational (never in the oracle). NOT v2.0.
- **H2. RA HUD completion** — **M.** Surface data RustyNES already decodes-then-drops: leaderboard scoreboard
  (#N of M), progress/challenge indicators, progress-bars/buckets/rarity, hardcore **pause-gating** (`rc_client_can_pause`).
  Source: vendored rcheevos `rc_client.h`. Low risk.
- **H3. Audio depth** — **M.** Stereo panning, reverb/crossfeed, output **device picker**, 20-band EQ, per-context
  volume. All in the **frontend mixer** — the core sample stream stays byte-identical. NOT v2.0.
- **H4. Per-game `<rom>.json` config overrides + DIP editor + lag counter** — **M.** Architectural keystone for
  per-game settings (Mesen2). Layers on the v1.2.0 game-DB; the frontend overlay, never the core harness. NOT v2.0.
- **H5. i18n framework** — **M.** RustyNES's one systemic gap — no localization anywhere. A string-catalog layer +
  the egui plumbing. Frontend. NOT v2.0.
- **H6. Web / wasm parity** — **M.** Lua-in-browser (the piccolo hooks are no-ops today), File System Access API,
  Gamepad API, PWA/offline + base64 `?settings=` share-links. NOT v2.0.
- **H7. Tier-1 performance (measure-first, byte-identical)** — **S–M.** Unified-DMA cycle fast-path (parallels the
  proven mapper-caps win, `full_frame` +3–5%) + a safe BLEP phase-row cache (`spectral` +2–5%). Adopt only on the
  **>3% Criterion-stable + byte-identical** bar. Exercise the existing PGO/BOLT gate.
- **H8. Spectator netplay** — **M.** A determinism-safe read-only extension of the rollback stack. Pairs with the
  live-netplay host/TURN matrix verify carryover (`deploy/`, `docs/netplay-webrtc.md` §4 → Verified) + adaptive
  input buffer + auto config-resync. NOT v2.0.
- **H9. Power-user niceties** — **S–M each.** Virtual Pad (clickable on-screen controller → `SharedInput`), input
  **Macros**/templates (feeds piano-roll pattern-paint), **BasicBot** (savestate-anchored brute-force search; pairs
  with B1 + `frameadvance`), movie subtitles → `.srt` export, multi-monitor/detachable tool windows (egui
  multi-viewport), A/V dump-options depth (codec/sync), Firmware Manager (FDS BIOS hash-verify/resolve — supports
  v1.6.0 FDS-proper), Multi-Disk Bundler (named FDS/multicart slots), Batch Runner (consolidate the ad-hoc per-mapper
  screenshot/movie-verify scripts into a first-class headless mode), Game Genie encoder UI + `.tbl` text tables.

## Out of scope / deferred to v2.0 (the timebase major)

The **one-clock + every-cycle-bus-access timebase rewrite** (ADR 0002) and everything it gates: the `$2002`
NMI-suppression race, the `apu_reset` `$4017` half-cycle (`apu_reset` FAIL #3 ×2), `mmc3_test_2/4` #3 (the 1-cycle
bracket), exact PAL 3.2:1 fractional alignment, the sprite-0 stale-shifter; the **CPU-multiplier overclock** (distinct
from F3's dot-resolution scanline insert); and **full Vs. DualSystem dual-core**. See the v2.0.0 plan + research dive.

## Maintainer decisions (researched default in **bold**)

- **Editing-tool writeback aggressiveness:** **full editors (A1–A3) gated like cheats** vs read-only-inspect-only-this-release.
- **IPC security posture:** **host-mediated `comm.*` behind off-by-default `script-ipc`, disabled in replay/netplay/RA**
  vs defer IPC entirely to a later release.
- **Browser RA:** **land fully in v1.7.0 (beta.5) incl. the manual live verify** vs keep scaffolding-only and defer.
- **SQLite (E3):** **`userdata` KV now, SQLite optional/later** vs both now.
- **Codename:** **"Forge"** vs the breadth-themed alternatives (Compendium / Archive / Reach / Ensemble).
- **Release size management:** five betas as above vs splitting beta.5 (reach/polish) into its own point release if
  the train runs long.
