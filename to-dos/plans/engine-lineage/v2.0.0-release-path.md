# Plan — RustyNES_v2 v2.0.0 release path

## Context

RustyNES v2 shipped **v1.0.0 final** on 2026-05-23 — AccuracyCoin 90.65%,
545 strict + 5 ignored, 60-ROM commercial-ROM oracle 60/60. The release
explicitly defers a known residual set to a v1.x/v2.0 horizon:

- **4 C1 IRQ-timing residuals** (`cpu_interrupts_v2/{2,3,5}` +
  `mmc3_test_2/4` sub-test #3). 17 independent fix attempts have been
  rolled back across sessions. Session-29's empirical finding
  (`docs/audit/session-29-c1-axis-final-conclusion-2026-05-23.md`):
  the residual is a fixed +2 PPU dot offset rooted in Mesen2's
  master-clock reset sequence (`Core/NES/NesCpu.cpp` lines 138/158:
  `_state.CycleCount = (uint64_t)-1` then `_masterClock += cpuDivider
  + cpuOffset` pre-loop, advancing PPU by ~2 dots without incrementing
  CycleCount). A global PPU shift cannot close it because every
  PPU-dependent baseline is calibrated to the current alignment.
  Closure requires the master-clock-precise scheduling refactor —
  replacing the integer-PPU-dot-per-CPU-cycle model with Mesen2's
  fractional 12-master-clocks-per-CPU-cycle model with explicit
  `StartCpuCycle → MemoryRead → EndCpuCycle` ordering.
- **13 AccuracyCoin residuals besides C1** (out of 139 assigned tests):
  3 sprite-eval residuals, 5 PPU misc residuals, 1 Implied Dummy
  Reads + DMC cascade, 4 APU edge cases (partially overlap with the
  catalog).
- **VRC7 OPLL FM audio** (mapper 85; `mix_audio` returns 0). ADR-0004
  deferred because no Rust OPLL crate is published; `emu2413` (MIT,
  `digital-sound-antiques/emu2413`) is the canonical reference C
  source, multi-week port to pure Rust.
- **FDS (Famicom Disk System)** — never started. nesdev_wiki has
  comprehensive docs (`FDS.md`, `FDS_BIOS.md`, `FDS_audio.md`,
  `FDS_disk_format.md`); Mesen2 implementation at
  `Core/NES/Mappers/FDS/`.
- **WebAssembly / mobile builds** — chip stack already `no_std + alloc`
  compatible; the frontend (`crates/nes-frontend`, `winit + wgpu + cpal
  + egui`) needs a wasm32 backend.
- **Netplay** — Mesen2 reference at `Core/Netplay/`; the determinism
  contract (CLAUDE.md "same seed + ROM + input ⇒ bit-identical") is
  the load-bearing prerequisite and is already met.
- **TAS movie recording/playback** — Mesen2 reference at `Core/Movie/`;
  same determinism prerequisite.
- **6 ignored commercial ROMs** (mapper-026 VRC6b pair structurally
  shares one bug + 4 others).
- **macOS x86_64 sunset** — GitHub Actions `macos-15-intel` runner
  retires August 2027; release workflow needs aarch64-only migration.
- **`gilrs` gamepad rebinding UI** — v1.0 ships hardcoded Xbox-style.

The user mandate: reach **v2.0.0** with all of the above fixed/added,
shipped through **incremental dev/test cycles** with **intermediate
v1.x tags** at logical milestones, using **continuous external research
on nesdev_wiki/ and the web** throughout.

This plan establishes that incremental path. v1.x releases are
SemVer-compatible additive improvements; v2.0.0 ships the breaking
master-clock refactor + everything saved up for that major bump (FDS,
netplay, save-state v5 format).

### v2.0.0 — Master-clock refactor + FDS + Netplay — 10-14 weeks

The major-version break. Three large work-streams in parallel-feasible
order.

**Sprint A — Master-clock-precise scheduling refactor (closes C1).
4-6 weeks.** Replace integer-PPU-dot-per-CPU-cycle scheduling with
Mesen2's fractional 12-master-clocks-per-CPU-cycle (NTSC), 16 (PAL),
15 (Dendy). Per Session-29's empirical conclusion this is the only
path to closure for the 4 C1 residuals.

- **A1.** New `MasterClock` type on `nes-core` (`u64` wrapping; saturating
  arithmetic). `Bus` trait gains `tick_master_clocks(n: u64)` replacing
  `tick_one_dot` / `tick_one_cpu_cycle`. CPU advances every 12 / 16 /
  15 master clocks (region-dispatched); PPU advances every 4 master
  clocks (NTSC) / 5 PAL / 5 Dendy (PAL/Dendy PPU runs slightly slower
  per nesdev master-clock page).
- **A2.** CPU read/write becomes `StartCpuCycle (advance pre-access
  ticks) → MemoryRead/Write → EndCpuCycle (advance remaining)`
  matching Mesen2's `NesCpu.cpp` StartCpuCycle/MemoryRead/EndCpuCycle.
  The cpu-c1-attempt-17-access-reorder feature scaffold that landed
  in v1.0.0 (`crates/nes-cpu/src/scheduler.rs`) is the foundation.
- **A3.** Reset sequence: replace integer 8-cycle reset advance with
  the Mesen2 pre-loop pattern (`_masterClock += cpuDivider + cpuOffset`
  pre-loop, then 8 cycles). Closes the +2 PPU dot offset.
- **A4.** Save-state v5: master-clock counter replaces independent
  `cpu_cycle` + `ppu_dot` + `apu_cycle` counters. ADR-0003 migration
  policy: v4 saves load via on-the-fly conversion (cpu_cycle×12 →
  master_clock).
- **A5.** Regenerate ALL PPU-dependent baselines under user
  authorisation: 60-ROM commercial-ROM oracle insta snapshots,
  81-PNG visual baseline corpus, audio_db hashes, 6 golden IRQ traces,
  `vbl_race_window_2002_read_sweep` predicate.
- **A6.** Flip the 4 C1 `#[ignore]`'d strict probes
  (`cpu_interrupts_v2/{2,3,5}` + `mmc3_test_2/4` #3) to strict-pass.
  Delete the `*_currently_fails` companions.
- **A7.** Side-benefit AccuracyCoin closures from the master-clock
  refactor: `$2002 flag timing` (per Session-27 audit, on the C1 axis)
  + any latent dot-precision tests that improve under fractional
  scheduling. Target: AccuracyCoin ≥ 99% (138/139), realistically 100%
  bar the one hardest edge case.

**Sprint B — FDS (Famicom Disk System). 3-4 weeks.** Mapper 20 +
disk file format + BIOS load + FDS audio.

- **B1.** `.fds` file format parser (header + side data;
  nesdev `FDS_file_format.md` / `FDS_disk_format.md`). Lands as
  `crates/nes-mappers/src/fds.rs` (NEW; ~2000 LoC including audio).
- **B2.** FDS BIOS load — user supplies `disksys.rom` (Nintendo
  copyright; never committed). Frontend prompts for path on first FDS
  ROM load; persist path in config.
- **B3.** Disk side switching UI (debugger overlay button + hotkey).
- **B4.** FDS audio — single-channel 64-step wavetable + LFO modulation
  + master volume (nesdev `FDS_audio.md`; Mesen2 `Core/NES/Mappers/FDS/FdsAudio.cpp`
  structural reference). Mixed via `mapper-audio` feature flag.
- **B5.** Snapshot of FDS-specific state in save-state v5 (disk side
  pointer, RAM image).

**Sprint C — Netplay foundation. 3-4 weeks.** Deterministic rollback
netcode leveraging the bit-identical determinism contract.

- **C1.** P2P session protocol — UDP hole-punched with WebRTC data
  channel fallback for browsers (the v1.3 wasm32 target). 60 Hz frame
  exchange with input-prediction + rollback. Mesen2's `Core/Netplay/`
  is GPL-3.0 structural reference only.
- **C2.** Lobby / matchmaking — code-based (4-character session code)
  via a thin signalling server (NEW `crates/netplay-signaling`;
  Rust + WebSocket; can be self-hosted or deployed to fly.io). Code
  + plan only; deployment is post-v2.0.
- **C3.** In-app session UI (host / join modal; in-game lag indicator;
  desync detection via per-frame CRC).
- **C4.** Determinism stress harness — 10,000-frame replay diff under
  100ms artificial latency injection.

**Sprint D — v2.0.0 final tag.** Workspace version bump 1.x → 2.0.0;
CHANGELOG `[2.0.0]` section; docs/STATUS.md rewrite; ROADMAP archives
Phase 6 / opens v2.x phase; pre-tag 10-gate gauntlet (one gate now
covers wasm32 too, per v1.3); git tag + push.

**Critical files (v2.0):** `crates/nes-core/src/scheduler.rs` (NEW),
`crates/nes-cpu/src/cpu.rs` (read1/write1 → StartCpuCycle/MemoryRead/
EndCpuCycle), `crates/nes-core/src/bus.rs` (Bus trait master-clock
API), `crates/nes-ppu/src/ppu.rs` (`tick(master_clocks)`),
`crates/nes-apu/src/apu.rs` (`tick(master_clocks)`),
`crates/nes-mappers/src/fds.rs` (NEW),
`crates/nes-mappers/src/fds_audio.rs` (NEW),
`crates/nes-core/src/netplay.rs` (NEW),
`crates/netplay-signaling/` (NEW workspace member),
`crates/nes-core/src/snapshot.rs` (v5),

**Verification (v2.0):** all 10 v1.0 gauntlet gates + 4 new: (11) C1

## Risk register

| Risk | Mitigation |
|---|---|
| OPLL port (v1.1) takes longer than 2-3 weeks (`emu2413` is ~3 KLOC dense C) | Time-box to 4 weeks; if exceeded, ship v1.1 with VRC7 silent (current state) and re-scope OPLL into v1.2 |
| Sprint 2.2 BG-pipeline change cascades into commercial-ROM oracle (Cascade A surface) | Re-baseline authorisation requested up-front; sacred-trio bisect after each sub-step |
| wasm32 cpal alternative not stable at v1.3 time | Fall back to web-audio direct (Worklet); accept slightly worse latency until cpal-web matures |
| Master-clock refactor (v2.0 Sprint A) is multi-week and breaks every chip's tick path | Land under a parallel `master-clock` cargo feature (default OFF) for ~2 weeks of parallel-implementation equivalence harness before flipping default; all 17 prior C1 rollback attempts are the proof this needs the full refactor not a surgical patch |
| FDS BIOS user-supply UX creates friction | Document clearly in README + frontend prompt; ship a "BIOS missing → load any FDS demo ROM to test" path |
| Netplay desync from a forgotten non-deterministic source | The determinism contract is already enforced by the per-frame CRC harness; netplay adds another sentinel (per-frame CRC over the wire) |
| v2.0.0 calendar slips past 6 months | Each v1.x is independently releasable and useful; if v2.0 slips, v1.5 is the LTS until v2.0 lands. Sprint A (master-clock) is the critical-path item — if it stalls, v2.0 ships with FDS + Netplay deferred to v2.1 |
| GPL-3.0 contamination from Mesen2 references | All Mesen2 reading is structural only. emu2413 (v1.1) is MIT — direct port is safe. FDS + Netplay structural patterns are public domain (nesdev wiki) + clean-room reimplementation. Per-file license headers verified at port time |
