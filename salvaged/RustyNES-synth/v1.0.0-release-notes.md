## RustyNES v1.0.0 — the cycle-accurate release

RustyNES 1.0.0 replaces the emulation core with a **cycle-accurate** rewrite that
meets the Mesen2 / higan / ares accuracy bar, and wraps it in a polished desktop +
browser frontend. This is a ground-up leap from the 0.8.x line.

### Highlights

- **Accuracy:** AccuracyCoin **100% (139/139)**, nestest **0-diff**, PPU-dot lockstep
  scheduling (the PPU is the master clock), sub-instruction PPU events, a band-limited
  audio mixer. NTSC / PAL / Dendy region timing.
- **Compatibility:** **51 mapper families** covering the vast majority of the commercial
  library; the full **Famicom Disk System** (real-BIOS); **Vs. System** + **PlayChoice-10**
  (RGB PPU palettes).
- **Modern features:** **netplay** (GGPO-style rollback over UDP + a browser WebRTC mesh,
  2–4 players), **RetroAchievements** (opt-in, native), **TAS movie** record/replay,
  save-states, rewind, **run-ahead** latency reduction, display-sync pacing with a
  lock-free audio path + dynamic rate control, and a dedicated emulation thread.
- **Desktop UX:** a menu bar, recent-ROMs, a first-run welcome, a tabbed Settings window,
  light/dark/system themes, an 8:7 pixel-aspect toggle, a status bar, and a full egui
  debugger (CPU / PPU / APU / memory / OAM / mapper panels).
- **WebAssembly:** a playable in-browser build deployed to GitHub Pages.

### Stack

Pure Rust — `winit` + `wgpu` + `egui` + `cpal`. A one-directional crate workspace
(`rustynes-cpu/ppu/apu/mappers/core/frontend` + `rustynes-netplay/cheevos/test-harness`),
`no_std`-capable chip stack, and a hard determinism contract (same seed + ROM + inputs
⇒ bit-identical framebuffer and audio).

### Notes

- This release integrates the cycle-accurate engine developed across the upstream
  "RustyNES v2" line; the `CHANGELOG.md` documents that lineage as the documentary
  `v0.9.0–v0.9.7` stages leading into this 1.0.0 production cut.
- **No commercial ROMs are distributed.** The accuracy oracle uses user-supplied dumps;
  only freely-redistributable test ROMs are committed.

### Build from source

```
cargo run --release -p rustynes-frontend -- path/to/rom.nes
```

Linux needs `libxkbcommon`, `wayland`, `alsa-lib`, `systemd-libs` (udev). See the README
for per-platform details and controls.
